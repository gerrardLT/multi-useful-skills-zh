# superpowers-中文 目录结构说明

## 项目定位

`superpowers-中文` 是一个面向 AI 编码代理的软件开发方法论仓库。核心是围绕 brainstorming、计划编写、调试、测试、并行代理、收尾验收等开发流程提供一组可组合 skill。

## 顶层目录与文件

- `README.md`
  - 仓库总入口。
- `AGENTS.md`、`CLAUDE.md`、`GEMINI.md`
  - 不同代理环境下的协作规则。
- `RELEASE-NOTES.md`
  - 版本发布记录。
- `CODE_OF_CONDUCT.md`
  - 社区规则。
- `package.json`
  - 依赖与脚本入口。
- `gemini-extension.json`
  - Gemini 扩展元数据。
- `agents/`
  - 代理角色说明。
- `commands/`
  - 简化命令入口。
- `docs/`
  - 平台说明、设计稿、计划文档。
- `hooks/`
  - 会话启动与注入逻辑。
- `scripts/`
  - 同步与版本脚本。
- `skills/`
  - 仓库核心。
- `tests/`
  - 触发测试、集成测试、脚本测试。
- `assets/`
  - 图标等静态资源。

## Skills 简要说明

- `using-superpowers`
  - 对话一开始就先建立 skill 使用方式和调用规则。
- `brainstorming`
  - 在任何创造性实现前先做需求、意图和设计探索。
- `writing-plans`
  - 在写代码前先写实现计划。
- `executing-plans`
  - 按既有实现计划推进执行。
- `test-driven-development`
  - 先写失败测试，再写实现。
- `systematic-debugging`
  - 先查根因，再提修复方案。
- `dispatching-parallel-agents`
  - 把彼此独立的任务并行分给多个 agent。
- `subagent-driven-development`
  - 在当前会话里按任务拆分执行一份实现计划。
- `requesting-code-review`
  - 完成后主动请求代码评审。
- `receiving-code-review`
  - 收到评审意见后先澄清、验证，再决定是否采纳。
- `verification-before-completion`
  - 声称完成前先跑验证并拿到证据。
- `finishing-a-development-branch`
  - 在工作完成后决定合并、提 PR 还是清理分支。
- `using-git-worktrees`
  - 为隔离开发创建安全的 git worktree。
- `writing-skills`
  - 创建、编辑和验证 skill 本身。

## 目录关系

- `skills/` 是方法论主体。
- `commands/` 是简化调用入口。
- `hooks/` 是会话注入层。
- `agents/` 是角色提示词层。
- `docs/` 是平台说明和设计沉淀层。
- `tests/` 负责验证触发逻辑和开发流程。
- `scripts/` 负责版本与插件同步。

## 建议阅读顺序

1. `README.md`
2. `CLAUDE.md` 或 `AGENTS.md`
3. `skills/using-superpowers/SKILL.md`
4. `skills/brainstorming/SKILL.md`
5. `skills/writing-plans/SKILL.md`
6. `skills/subagent-driven-development/SKILL.md`
7. `docs/README.codex.md` 或其他平台文档

## 常见工作路径

- 想理解整套方法论：
  - 看 `README.md` 和 `skills/using-superpowers/`
- 想改 brainstorming：
  - 看 `skills/brainstorming/` 和 `scripts/`
- 想改计划编写或执行流程：
  - 看 `skills/writing-plans/`、`skills/executing-plans/`
- 想改调试或测试方法：
  - 看 `skills/systematic-debugging/`、`skills/test-driven-development/`
- 想改代理协作流：
  - 看 `skills/subagent-driven-development/`、`skills/dispatching-parallel-agents/`
- 想验证改动：
  - 看 `tests/`

# superpowers-中文 目录结构说明

## 项目定位

`superpowers-中文` 是一个面向 AI 编码代理的软件开发方法论仓库。它围绕 brainstorming、写计划、TDD、调试、子代理协作、完成收尾等开发流程提供一组可组合 skill。

## 顶层目录说明

- `.claude-plugin/`、`.codex/`、`.codex-plugin/`、`.cursor-plugin/`、`.opencode/`
  - 各平台的接入或插件发布相关内容。
- `.github/`
  - GitHub 工作流与仓库配置。
- `agents/`
  - 代理角色提示词或代理定义。
- `assets/`
  - 图标、视觉资源。
- `commands/`
  - 命令层入口，例如把 skill 以更直接的命令形式暴露。
- `docs/`
  - 使用说明、测试说明、设计规格、计划文档、平台说明。
- `hooks/`
  - 会话启动、环境注入、跨平台 hook 脚本。
- `scripts/`
  - 版本同步、构建、发布等脚本。
- `skills/`
  - 仓库核心。
  - 每个 skill 一个目录，通常包含 `SKILL.md`、参考文档、模板或辅助脚本。
- `tests/`
  - 各类测试，包括平台测试、技能触发测试、子代理开发测试等。

## 关键文件说明

- `README.md`
  - 项目主入口与整体使用方式。
- `AGENTS.md`
  - 面向代理的仓库协作说明。
- `CLAUDE.md`
  - 对贡献和行为的高约束规则说明。
- `GEMINI.md`
  - Gemini 相关接入说明。
- `RELEASE-NOTES.md`
  - 版本发布说明。
- `CODE_OF_CONDUCT.md`
  - 社区行为准则。
- `package.json`
  - 依赖、脚本、包配置。
- `gemini-extension.json`
  - Gemini 扩展元数据。

## 目录关系

- `skills/` 是方法论主体。
  - 真正定义如何 brainstorming、写计划、调试、验证、协作。
- `commands/`
  - 是更轻量的调用入口，通常与 `skills/` 配合使用。
- `docs/`
  - 负责解释方法、平台差异、设计决策、演进计划。
- `hooks/`
  - 负责在实际代理会话中注入上下文或自动执行前置逻辑。
- `agents/`
  - 存放被 skill 引用的代理角色说明。
- `tests/`
  - 用来验证 skill 行为、平台兼容性和协作流程。
- `scripts/`
  - 支撑打包、同步与发布。

## 建议阅读顺序

1. `README.md`
2. `CLAUDE.md`
3. `skills/using-superpowers/SKILL.md`
4. `skills/brainstorming/`、`skills/writing-plans/`、`skills/subagent-driven-development/`
5. 需要平台细节时看 `docs/README.codex.md`、`docs/README.opencode.md` 等
6. 需要运行机制时看 `hooks/`、`agents/`
7. 需要验证或维护时看 `tests/`、`scripts/`

## 常见工作路径

- 想理解这套方法论怎么用：
  - 先看 `README.md` 和 `skills/using-superpowers/`
- 想看完整开发流程：
  - 依次看 `brainstorming` -> `writing-plans` -> `subagent-driven-development` -> `verification-before-completion` -> `finishing-a-development-branch`
- 想改某个 skill：
  - 直接进入 `skills/<skill-name>/`
- 想看平台差异：
  - 看 `docs/` 与对应插件目录
- 想看自动注入或启动逻辑：
  - 看 `hooks/`

# Product-Manager-Skills-中文 目录结构说明

## 项目定位

`Product-Manager-Skills-中文` 是一个面向产品经理工作流的技能仓库。核心内容不是应用代码，而是：

- `skills/` 里的 PM 技能
- `commands/` 里的调用入口
- `docs/` 里的平台接入与使用说明
- `scripts/` 里的构建、校验、打包脚本

## 顶层目录与文件

- `README.md`
  - 仓库总入口，先看这个。
- `START_HERE.md`
  - 新用户快速上手。
- `PLANS.md`
  - 仓库后续规划。
- `MARKETPLACE_STRATEGY.md`
  - 分发与上架策略。
- `AGENTS.md`、`CLAUDE.md`
  - 面向代理协作的规则。
- `CONTRIBUTING.md`
  - 贡献规范。
- `app/`
  - 一个轻量应用层，目前主要是 Streamlit 入口。
- `catalog/`
  - 技能与命令索引。
- `commands/`
  - 把多个 skill 封装成更直接的命令入口。
- `docs/`
  - 各平台接入文档、安装文档、运维文档。
- `research/`
  - 方法论、背景材料、专题资料。
- `scripts/`
  - 构建、检查、打包、查找工具。
- `skills/`
  - 仓库最核心的内容。

## Skills 简要说明

### 战略与定位

- `acquisition-channel-advisor`
  - 评估获客渠道该扩大、继续测试还是停止。
- `business-health-diagnostic`
  - 从增长、留存、效率和资本使用诊断 SaaS 健康度。
- `company-research`
  - 生成公司研究简报，用于面试、竞品或市场研究。
- `feature-investment-advisor`
  - 评估某个功能是否值得继续投入。
- `finance-based-pricing-advisor`
  - 从财务角度评估定价调整是否应上线。
- `finance-metrics-quickref`
  - 快速查询 SaaS 财务指标、公式和 benchmark。
- `pestel-analysis`
  - 分析外部宏观环境对产品和战略的影响。
- `positioning-statement`
  - 生成 Geoffrey Moore 风格定位陈述。
- `positioning-workshop`
  - 运行定位工作坊，梳理客户、需求、品类和差异化。
- `prioritization-advisor`
  - 帮你选择合适的优先级框架。
- `product-strategy-session`
  - 跑完整产品战略 session，覆盖定位、discovery 和 roadmap。
- `recommendation-canvas`
  - 评估 AI 产品想法是否值得投资或推荐。
- `roadmap-planning`
  - 把战略转成可执行的 roadmap。
- `saas-economics-efficiency-metrics`
  - 评估 SaaS 单位经济和资本效率。
- `saas-revenue-growth-metrics`
  - 计算 SaaS 收入、留存和增长指标。
- `tam-sam-som-calculator`
  - 估算 TAM、SAM、SOM。

### Discovery 与问题定义

- `context-engineering-advisor`
  - 判断你做的是 context stuffing 还是 context engineering。
- `customer-journey-map`
  - 生成客户旅程图。
- `customer-journey-mapping-workshop`
  - 运行客户旅程映射工作坊。
- `discovery-interview-prep`
  - 规划客户发现访谈。
- `discovery-process`
  - 跑完整 discovery 流程。
- `epic-hypothesis`
  - 把 epic 写成可验证假设。
- `jobs-to-be-done`
  - 用 JTBD 格式梳理用户任务、痛点和收益。
- `lean-ux-canvas`
  - 引导团队完成 Lean UX Canvas。
- `opportunity-solution-tree`
  - 构建 Opportunity Solution Tree。
- `problem-framing-canvas`
  - 用 Problem Framing Canvas 先把问题定义清楚。
- `problem-statement`
  - 写出以用户为中心的问题陈述。
- `proto-persona`
  - 创建轻量 proto-persona。

### 产出物与交付拆解

- `eol-message`
  - 撰写下线公告。
- `epic-breakdown-advisor`
  - 把 epic 拆成可交付的 user stories。
- `prd-development`
  - 生成结构化 PRD。
- `press-release`
  - 撰写 Amazon 风格新闻稿。
- `storyboard`
  - 生成六格 storyboard。
- `user-story`
  - 创建带验收标准的 user story。
- `user-story-mapping`
  - 创建用户故事地图。
- `user-story-mapping-workshop`
  - 运行用户故事映射工作坊。
- `user-story-splitting`
  - 把大 story 或 epic 拆小。

### 职业成长与领导力

- `ai-shaped-readiness-advisor`
  - 评估团队是否已进入 AI-shaped 阶段。
- `altitude-horizon-framework`
  - 用 altitude/horizon 诊断 PM 到 Director 的转型缺口。
- `director-readiness-advisor`
  - 辅导 PM 向 Director 转型。
- `executive-onboarding-playbook`
  - 规划 VP/CPO 的 30-60-90 天入职路径。
- `product-sense-interview-answer`
  - 组织 PM product-sense 面试答案。
- `vp-cpo-readiness-advisor`
  - 辅导 VP 或 CPO 层级转型。

### 工作坊与技能维护

- `pol-probe`
  - 设计生命证明（PoL）探测，用于验证关键假设。
- `pol-probe-advisor`
  - 选择合适的 PoL 探测方式。
- `skill-authoring-workflow`
  - 把原始 PM 内容整理成合规、可发布的 skill。
- `workshop-facilitation`
  - 为交互式 workshop 提供统一主持节奏。

## 目录关系

- `skills/` 是能力定义层。
- `commands/` 是能力调用层。
- `catalog/` 是索引层。
- `docs/` 是接入与说明层。
- `scripts/` 是构建与校验层。
- `research/` 是方法论支撑层。
- `app/` 是演示或交互入口层。

## 建议阅读顺序

1. `README.md`
2. `START_HERE.md`
3. `docs/Using PM Skills 101.md`
4. `skills/<skill-name>/SKILL.md`
5. `commands/README.md`
6. `catalog/README.md`
7. `scripts/README.md`

## 常见工作路径

- 想直接用某个技能：
  - 看 `skills/<skill-name>/SKILL.md`
- 想找现成命令：
  - 看 `commands/` 和 `catalog/commands.md`
- 想接入某个平台：
  - 看 `docs/Using PM Skills with *.md`
- 想增加或修改技能：
  - 先改 `skills/`
  - 再同步 `catalog/`、`commands/`、`docs/`
- 想做构建或打包：
  - 看 `scripts/README.md` 和相关 `build-*` 脚本

# Product-Manager-Skills-中文 目录结构说明

## 项目定位

`Product-Manager-Skills-中文` 是一个面向产品经理与 AI agent 协作的技能仓库，核心内容是 PM skill、workflow command、平台使用文档、研究资料和辅助脚本。

## 顶层目录说明

- `.claude-plugin/`
  - Claude 插件相关发布内容。
- `.github/`
  - GitHub 自动化、模板或工作流配置。
- `app/`
  - 应用层内容。
  - 当前主要包括 Streamlit 界面与其相关说明。
- `catalog/`
  - 技能目录、命令索引与自动生成清单。
- `commands/`
  - 面向工作流的命令封装。
  - 通常是把 skill 组合成更直接可用的调用入口。
- `docs/`
  - 平台接入说明、安装指南、公告、运营文档。
- `research/`
  - 背景研究、框架来源、专题资料。
- `scripts/`
  - 查找、校验、打包、生成目录等辅助脚本。
- `skills/`
  - 仓库核心内容。
  - 每个子目录对应一个 PM skill，通常包含 `SKILL.md`、模板、示例或脚本。

## 关键文件说明

- `README.md`
  - 仓库总入口，适合先读。
- `START_HERE.md`
  - 新用户快速起步路径。
- `PLANS.md`
  - 仓库演进计划与阶段说明。
- `MARKETPLACE_STRATEGY.md`
  - 上架与分发策略说明。
- `AGENTS.md`
  - 面向代理的协作规则。
- `CLAUDE.md`
  - Claude 环境下的维护与行为要求。
- `CONTRIBUTING.md`
  - 贡献规范。

## 目录关系

- `skills/` 是核心能力层。
  - 定义产品经理可复用的分析、写作、规划、发现、优先级和工作流能力。
- `commands/` 是调用封装层。
  - 把 `skills/` 中的能力包装成更直接的命令入口。
- `docs/` 是接入与使用层。
  - 说明这些 skill 如何在 Claude、Codex、Windsurf、n8n、OpenClaw 等平台中使用。
- `catalog/` 是索引层。
  - 帮助人或工具发现有哪些 skill、command。
- `scripts/` 是维护工具层。
  - 用于生成、校验、打包和查找。
- `research/` 是知识来源层。
  - 用来解释 skill 背后的框架和方法。
- `app/` 是交互式产品层。
  - 把仓库内容转成可直接使用的界面。

## 建议阅读顺序

1. `README.md`
2. `START_HERE.md`
3. `docs/Using PM Skills 101.md`
4. `skills/` 中目标 skill 的 `SKILL.md`
5. 需要命令式使用时看 `commands/`
6. 需要平台接入时看 `docs/`
7. 需要维护仓库时看 `scripts/`、`catalog/`、`research/`

## 常见工作路径

- 想直接使用某个 PM skill：
  - 先看 `skills/<skill-name>/SKILL.md`
- 想要现成工作流命令：
  - 看 `commands/`
- 想在具体 AI 平台里接入：
  - 看 `docs/`
- 想查有哪些 skill：
  - 看 `catalog/` 或用 `scripts/`
- 想改仓库能力：
  - 先改 `skills/`
  - 如有必要，再同步 `commands/`、`catalog/`、`docs/`

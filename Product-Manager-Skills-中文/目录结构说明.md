# Product-Manager-Skills-中文 目录结构说明

## 项目定位

`Product-Manager-Skills-中文` 是一个面向产品经理工作流的技能仓库。核心内容不是应用代码，而是：

- `skills/` 里的 PM 技能
- `commands/` 里的调用入口
- `docs/` 里的平台接入与使用说明
- `scripts/` 里的构建、校验、打包脚本

## 顶层目录与文件

- `README.md`
  - 仓库总入口，先看这个。
- `START_HERE.md`
  - 新用户快速上手。
- `PLANS.md`
  - 仓库后续规划。
- `MARKETPLACE_STRATEGY.md`
  - 分发与上架策略。
- `AGENTS.md`、`CLAUDE.md`
  - 面向代理协作的规则。
- `CONTRIBUTING.md`
  - 贡献规范。
- `app/`
  - 一个轻量应用层，目前主要是 Streamlit 入口。
- `catalog/`
  - 技能与命令索引。
- `commands/`
  - 把多个 skill 封装成更直接的命令入口。
- `docs/`
  - 各平台接入文档、安装文档、运维文档。
- `research/`
  - 方法论、背景材料、专题资料。
- `scripts/`
  - 构建、检查、打包、查找工具。
- `skills/`
  - 仓库最核心的内容。

## 目录关系

- `skills/` 是能力定义层。
- `commands/` 是能力调用层。
- `catalog/` 是索引层。
- `docs/` 是接入与说明层。
- `scripts/` 是构建与校验层。
- `research/` 是方法论支撑层。
- `app/` 是演示或交互入口层。

## 建议阅读顺序

1. `README.md`
2. `START_HERE.md`
3. `docs/Using PM Skills 101.md`
4. `skills/<skill-name>/SKILL.md`
5. `commands/README.md`
6. `catalog/README.md`
7. `scripts/README.md`

## 常见工作路径

- 想直接用某个技能：
  - 看 `skills/<skill-name>/SKILL.md`
- 想找现成命令：
  - 看 `commands/` 和 `catalog/commands.md`
- 想接入某个平台：
  - 看 `docs/Using PM Skills with *.md`
- 想增加或修改技能：
  - 先改 `skills/`
  - 再同步 `catalog/`、`commands/`、`docs/`
- 想做构建或打包：
  - 看 `scripts/README.md` 和相关 `build-*` 脚本

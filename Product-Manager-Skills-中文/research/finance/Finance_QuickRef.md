# 产品经理财务指标快速参考

**一页速查表，涵盖核心SaaS与产品财务指标**

---

| **指标** | **公式** | **衡量内容** | **产品经理为何关注** | **好/坏基准** |
|------------|-------------|----------------------|------------------|------------------------|
| **收入** | 费用前的总销售额 | 顶层收入 | 每个功能都应与收入挂钩（直接或间接） | 增长率比绝对数字更重要；取决于上下文 |
| **毛利率** | (收入 - 销货成本) / 收入 × 100 | 扣除直接成本后的收入百分比 | 高毛利功能远比低毛利功能更有价值 | SaaS：70-85%为佳；<60%需关注 |
| **CAC** | 总销售与营销支出 / 新客户数 | 获取一个客户的成本 | 决定整体市场进入策略和渠道决策 | 因模式而异：企业级$10K+可接受；中小企业<$500 |
| **LTV** | ARPU × 平均生命周期（或更复杂） | 来自一个客户的总收入 | 告知你能在获客上花费多少 | 必须为CAC的3倍以上；因细分市场而异 |
| **流失率** | 流失客户数 / 期初客户数 | 取消订阅的客户百分比 | SaaS的隐形杀手；削弱所有获客努力 | 月度：<2%优秀，5%+危机。年度：<10%优秀，>30%危机 |
| **ARPU** | 总收入 / 总用户数 | 每用户平均收入 | 每个客户贡献更多收入通常优于获取更多客户 | 因模式差异巨大；跟踪趋势比绝对值更重要 |
| **ARPA** | MRR / 活跃账户数 | 每账户平均收入 | 评估账户级定价和交易质量 | B2B SaaS：$100-$10K+，取决于细分市场 |
| **ARPA/ARPU** | 两个指标结合 | 账户价值 vs. 席位变现 | 防止因分母错误导致的打包失误 | 高ARPA + 低ARPU = 每席位变现不足 |
| **ACV** | 每份合同的年经常性收入 | 年化合同价值（不含一次性费用） | 比较不同合同结构的经济性 | 中小企业：$5K-$25K；中端市场：$25K-$100K；企业级：$100K+ |
| **MRR/ARR** | MRR × 12 = ARR | 可预测的经常性收入 | 订阅业务的心跳；估值倍数为5-10倍以上 | 增长率和质量很重要；跟踪组成部分（新增、扩展、流失） |
| **烧钱率** | 每月现金消耗 - 收入 | 每月消耗的现金 | 决定你能构建什么以及何时需要融资 | 净烧钱<$200K可控；>$500K需要清晰的收入路径 |
| **跑道** | 现金余额 / 月度烧钱率 | 资金耗尽前的月数 | 字面意义上的生存指标 | 12个月以上为佳；<6个月进入危机模式 |
| **运营支出** | 运营费用（销售与营销、研发、行政） | 运营业务的成本（非销货成本） | 你团队的薪资；“效率”削减发生的地方 | 随规模扩大，增长应慢于收入 |
| **净收入** | 收入 - 所有费用 | 实际利润/亏损 | 真正的底线；你赚钱了吗？ | 早期SaaS常为负；成熟期应有10-20%+利润率 |
| **LTV:CAC** | LTV / CAC | 单位经济效率 | 增长是可持续的还是在亏本买收入？ | 3:1健康；<1:1不可持续；>5:1可能投资不足 |
| **回收期** | CAC / (月度ARPU × 毛利率%) | 收回CAC所需的月数 | 现金效率；越快=越早可再投资 | <12个月优秀；12-18个月可接受；>24个月需关注 |
| **NRR** | (期初ARR + 扩展 - 流失 - 收缩) / 期初ARR | 收入留存 + 扩展 | 圣杯：无需新客户即可增长 | >120%卓越；100-120%良好；<90%有问题 |
| **贡献利润率** | (收入 - 所有可变成本) / 收入 | 扣除所有可变成本后的真实贡献 | 毛利率 + 支持/处理/可变成本 | SaaS中60-80%为佳；<40%需关注 |
| **扩展收入** | 追加销售 + 交叉销售 + 用量增长 | 来自现有客户的额外收入 | 资本效率最高的收入（无CAC） | 应驱动NRR >100%；目标占总收入的20-30% |
| **40法则** | 增长率% + 利润率% | 增长与效率的平衡 | 增长/盈利能力权衡的框架 | >40健康；25-40可接受；<25需关注 |
| **总收入 vs. 净收入** | 净收入 = 总收入 - 折扣 - 退款 - 信用额度 | 你实际保留的部分 | 折扣和退款可能隐藏糟糕的获客质量 | 退款率>10%是危险信号；按渠道跟踪 |
| **收入集中度** | 前N大客户 / 总收入 | 对最大客户的依赖 | 存在性风险和路线图扭曲 | 最大客户<10%；前10大<40%为理想 |
| **魔术数字** | (本季度收入 - 上季度收入) × 4 / 上季度销售与营销支出 | 销售与营销效率 | 你的GTM引擎有效吗？ | >0.75高效；0.5-0.75可接受；<0.5需在规模化前修复 |
| **速动比率 (SaaS)** | (新增MRR + 扩展MRR) / (流失MRR + 收缩) | 收入增长 vs. 损失 | 你是建立在坚实基础上还是在跑步机上奔跑？ | >4卓越；2-4健康；<2漏水桶 |
| **单位经济** | 每单位收入 - 每单位成本 | 每个“单位”的盈利能力 | 商业模式从根本上可行吗？ | 需要正贡献；目标在扣除所有可变成本后>$0 |
| **收入构成** | 按产品/细分市场分解的收入 | 投资组合构成 | 哪些产品为业务提供资金；投资方向 | 跟踪趋势；理想情况是单一产品不超过60% |
| **队列分析** | 按加入日期对客户分组；跟踪行为 | 业务随时间是在改善还是恶化 | 混合指标隐藏了关键趋势 | 近期队列表现应与旧队列相同或更好 |
| **细分市场回收期** | CAC / 月度贡献（按细分市场） | 按客户细分市场的回收期 | 不同细分市场有不同的经济性 | 跨细分市场比较以分配资源 |
| **增量性** | 由行动引起的收入 - 无论如何都会发生的情况 | 营销/促销的真实影响 | 不要庆祝无论如何都会发生的收入 | 尽可能通过留出测试来衡量 |
| **按渠道的CAC回收期** | CAC / 月度贡献（按获客渠道） | 哪些渠道最快收回投资 | 并非所有渠道生而平等 | 优化组合；不要只扩大最高流量的渠道 |
| **毛利率回收期** | CAC / (月度价格 × 毛利率%) | 使用实际利润而非收入计算的回收期 | 比简单的CAC回收期更准确 | 通常比简单回收期长1.5-2倍 |

## 快速决策框架

**评估功能投资时：**
1. **收入影响**：直接（定价、附加组件）还是间接（留存、转化）？
2. **毛利影响**：销货成本是多少？会稀释毛利吗？
3. **CAC影响**：会降低获客成本或启用新渠道吗？
4. **LTV影响**：会降低流失率或驱动扩展吗？
5. **权衡**：投资回报率是多少？收入/成本 或 LTV影响/投资？

**评估定价变更时：**
1. **ARPU/ARPA影响**：会增加每个客户的收入吗？
2. **转化影响**：会帮助还是损害试用到付费的转化？
3. **流失影响**：会造成流失风险还是降低风险？
4. **NRR影响**：会启用扩展还是造成收缩？
5. **CAC影响**：会改变回收期或渠道可行性吗？

**评估市场进入投资时：**
1. **CAC**：通过此渠道获取一个客户的成本是多少？
2. **LTV**：获取的客户是高质量的吗（留存、扩展）？
3. **回收期**：你多快能收回投资？
4. **魔术数字**：整体销售与营销支出效率如何（>0.75）？
5. **可扩展性**：此渠道能否支撑你所需的业务增长？

**评估业务健康状况时：**
1. **增长率**：对于你所处的阶段，增长是否足够快？
2. **NRR**：现有客户是在扩展还是收缩？
3. **速动比率**：你的增长是否大于损失？
4. **40法则**：你是否平衡了增长与效率？
5. **烧钱 vs. 跑道**：你能在现金耗尽前达到下一个里程碑吗？

## 需警惕的危险信号

| **危险信号** | **含义** | **应对措施** |
|-------------|-------------------|----------------|
| 流失率逐队列上升 | 产品市场契合度下降 | 停止扩大获客；先修复留存 |
| LTV:CAC <1.5 | 亏本买收入 | 在规模化前降低CAC或提高LTV |
| 回收期 >18个月 | 现金效率低的增长 | 提高转化率、降低CAC或提高定价 |
| NRR <90% | 客户在收缩，而非扩展 | 在投资获客前修复产品价值 |
| 魔术数字 <0.5 | 销售与营销支出效率极低 | 在增加支出前修复产品、ICP或销售流程 |
| 最大客户占收入 >30% | 存在性集中风险 | 立即多元化；不要只为他们构建 |
| 毛利率 <60% (SaaS) | 成本结构问题 | 提高价格、降低销货成本或重新考虑该功能 |
| 速动比率 <2 | 漏水桶问题 | 在注入更多（获客）前堵住漏洞（留存） |
| 退款率 >10% | 获客质量问题 | 修复信息传递、目标定位或引导流程 |
| 收入增长但利润率崩溃 | 无利可图的增长 | 按产品/细分市场评估利润率；削减低利润收入 |

## 产品经理常见陷阱

1. **优化错误指标**：CAC看起来很好，但LTV很糟糕 → 优化LTV:CAC，而非单独优化CAC
2. **庆祝虚荣指标**：收入增长50%，但流失也增长50% → 净进展为零
3. **忽略细分市场差异**：混合指标隐藏了你一半业务无利可图的事实
4. **混淆相关性与因果性**：新功能发布，收入上升 → 是功能的作用还是季节性？
5. **不跟踪队列**：在稳定的混合指标下，你的业务可能正在恶化
6. **过早规模化**：扩大一个低效的引擎（低魔术数字）只会更快地烧钱
7. **对留存投资不足**：获客感觉有成效；直到流失危机前，留存是隐形的
8. **忘记货币的时间价值**：5年后的$10K ≠ 今天的$10K
9. **平等对待所有收入**：来自1个客户的$1M ≠ 来自1,000个客户的$1M
10. **构建你最大客户想要的功能**：集中风险 + 你在构建定制软件，而非产品

## 应配对使用的指标（切勿单独使用）

- **CAC** → 始终与 **LTV** 配对（使用LTV:CAC比率）
- **收入增长** → 始终与 **利润率** 和 **烧钱率** 配对
- **流失率** → 始终与 **队列分析** 配对
- **MRR/ARR** → 始终与 **NRR** 配对（质量 > 数量）
- **毛利率** → 始终与 **贡献利润率** 配对（包含可变成本）
- **魔术数字** → 始终与 **CAC回收期** 配对（效率 + 现金时机）
- **NRR** → 始终与 **扩展 vs. 收缩分解** 配对
- **LTV** → 始终与 **回收期** 配对（总价值 + 现金时机）
- **单位经济** → 始终与 **CAC** 配对（每单位 + 获客）
- **收入构成** → 始终与 **按产品的利润率** 配对（构成 + 质量）

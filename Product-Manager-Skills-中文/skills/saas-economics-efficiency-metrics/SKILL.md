## 目的

确定您的SaaS业务模型是否基本可行且资本高效。用此技能计算单位经济指标、评估盈利能力、管理现金跑道，并决定何时扩张与优化。对于融资、董事会报告和做出明智的投资权衡至关重要。

这不是财务报表工具，而是一个给产品经理使用的分析框架，帮助你判断业务是否能支撑增长、何时应优先考虑效率而非增长，以及哪些投入真正能带来正回报。

## 关键概念

### 单位经济家族

衡量客户层面盈利能力的指标——可持续SaaS的基础。

**毛利率** — 收入扣除直接成本（COGS）后的剩余百分比。
- **为什么产品经理关心：** 一个以80%利润率带来100万美元收入的功能，远比以30%利润率带来100万美元收入的功能更有价值。利润率决定了应优先开发哪些功能。
- **公式：** `(收入 - COGS) / 收入 × 100`
- **COGS包括：** 托管、基础设施、支付处理、客户入职成本
- **基准：** SaaS 70-85% 良好；<60% 需关注

**CAC（客户获取成本）** — 获取一个客户的总成本。
- **为什么产品经理关心：** 塑造整个市场进入策略。决定了哪些渠道可行，以及在产品驱动增长上可以投入多少。
- **公式：** `总销售与营销支出 / 新获取客户数`
- **基准：** 因模型而异——企业级$10K+可接受；SMB目标<$500
- **包括：** 营销支出、销售人员工资、工具、佣金

**LTV（生命周期价值）** — 预期从一个客户在其生命周期内获得的总收入。
- **为什么产品经理关心：** 告诉你能在获客上花费多少。更高的LTV支持使用高端渠道和更长的回收期。
- **公式（简单）：** `ARPU × 平均客户生命周期（月）`
- **公式（更好）：** `ARPU × 毛利率 % / 流失率`
- **公式（高级）：** 考虑扩展、折现率、特定群组的留存率
- **基准：** 必须为CAC的3倍以上；因细分市场而异

**LTV:CAC比率** — 客户获取支出的效率。
- **为什么产品经理关心：** 增长是否可持续，还是你在亏本购买收入？决定了何时扩张与优化。
- **公式：** `LTV / CAC`
- **基准：** 3:1 健康；<1:1 不可持续；>5:1 可能投入不足
- **注意：** 仅此比率不能说明全部问题——还需考虑回收期

**回收期** — 从客户收入中收回CAC所需的月数。
- **为什么产品经理关心：** 现金效率。更快的回收期 = 更早再投资。即使LTV:CAC良好，缓慢的回收期也可能扼杀增长。
- **公式：** `CAC / (月度ARPU × 毛利率 %)`
- **基准：** <12个月 优秀；12-18个月 可接受；>24个月 需关注
- **关键：** 必须有现金来支撑回收期

**贡献毛利** — 扣除所有变动成本（不仅仅是COGS）后的收入。
- **为什么产品经理关心：** 真实的单位盈利能力。包括支持、处理费、变动运营费用。
- **公式：** `(收入 - 所有变动成本) / 收入 × 100`
- **变动成本：** COGS + 支持 + 支付处理 + 变动客户成功成本
- **基准：** SaaS 60-80% 良好；<40% 需关注

**毛利率回收期** — 使用实际利润而非收入计算的回收期。
- **为什么产品经理关心：** 比简单回收期更准确。显示真实的现金回收时间。
- **公式：** `CAC / (月度ARPU × 毛利率 %)`
- **基准：** 通常比简单收入回收期长1.5-2倍

**按渠道CAC回收期** — 比较不同获客渠道的回收期。
- **为什么产品经理关心：** 并非所有渠道都一样。根据回收期效率优化渠道组合。
- **公式：** 分别计算每个渠道的CAC和回收期
- **用途：** 当现金紧张时，将预算分配给回收期更快的渠道

### 资本效率家族

衡量你使用现金来发展业务的效率的指标。

**消耗率** — 每月消耗的现金。
- **为什么产品经理关心：** 决定了你能构建什么以及何时需要融资。高消耗率需要激进的收入增长。
- **公式（总消耗）：** `每月现金支出（所有费用）`
- **公式（净消耗）：`每月现金支出 - 每月收入`
- **基准：** 早期阶段净消耗<$200K可管理；>$500K需要清晰的收入路径

**跑道** — 现金耗尽前的月数。
- **为什么产品经理关心：** 生存指标。决定了里程碑、融资、盈利的时间表。
- **公式：** `现金余额 / 月度净消耗`
- **基准：** 12+个月 良好；6-12个月 可管理；<6个月 危机模式
- **规则：** 在跑道剩余6-9个月时融资，而不是3个月

**运营费用** — 运营业务的成本（不包括COGS）。
- **为什么产品经理关心：** 你的团队工资在这里。经济下行时“效率”削减发生的地方。
- **类别：** 销售与营销（S&M）、研发（R&D）、一般与行政（G&A）
- **基准：** 随着规模扩大，增长应慢于收入（经营杠杆）

**净利润（利润率）** — 扣除所有费用后的实际利润或亏损。
- **为什么产品经理关心：** 真实的底线。你在赚钱吗？你能自筹资金增长吗？
- **公式：** `收入 - 所有费用（COGS + 运营费用）`
- **基准：** 早期SaaS通常为负（增长模式）；成熟期应为10-20%+利润率

**营运资金影响** — 收入确认与现金收取之间的现金时间差。
- **为什么产品经理关心：** 预付的年度合同增加现金。月度账单延迟现金。影响跑道计算。
- **示例：** 100万美元年度合同预付 = 现在获得100万美元现金，而不是每月8.3万美元
- **用途：** 在规划跑道时理解现金与收入的时间差

---

### 效率比率家族

衡量增长与盈利能力权衡的复合指标。

**40法则** — 增长率 + 利润率应超过40%。
- **为什么产品经理关心：** 平衡增长与效率的框架。指导何时应优先考虑盈利能力而非增长。
- **公式：** `收入增长率 % + 利润率 %`
- **基准：** >40 健康；25-40 可接受；<25 需关注
- **示例：** 60%增长 + (-20%)利润率 = 40（健康的增长模式SaaS）
- **示例：** 20%增长 + 25%利润率 = 45（健康的成熟SaaS）

**魔术数字** — 销售与营销效率。
- **为什么产品经理关心：** 你的市场进入引擎有效吗？你应该扩大支出还是先优化？
- **公式：** `(本季度收入 - 上季度收入) × 4 / 上季度销售与营销支出`
- **基准：** >0.75 高效；0.5-0.75 可接受；<0.5 扩张前需修复
- **注意：** “× 4”将季度收入变化年化

**经营杠杆** — 收入增长与成本增长的比较。
- **为什么产品经理关心：** 你在高效扩张吗？收入增长应快于成本增长。
- **衡量：** 随时间推移的收入增长率与运营费用增长率
- **良好：** 收入增长50%，运营费用增长30%（正杠杆）
- **不佳：** 收入增长20%，运营费用增长40%（负杠杆）

**单位经济** — 衡量每个“单位”（客户、席位、交易）盈利能力的通用术语。
- **为什么产品经理关心：** 业务模型在单位层面是否根本可行？
- **计算：** 每单位收入 - 每单位成本
- **要求：** 需要正贡献；目标是在所有变动成本后 >$0

---

### 反模式（它不是什么）

- **不是虚荣指标：** 如果回收期需要4年而客户在3年时流失，高LTV毫无意义。
- **不是静态基准：** “良好”的CAC因业务模型（产品驱动增长 vs. 企业销售）而差异巨大。
- **不是孤立的数字：** 没有回收期的LTV:CAC比率可能产生误导（比率很好，现金效率极差）。
- **不仅仅是财务问题：** 产品经理必须掌握单位经济——每个功能决策都影响利润率和CAC。

---

### 何时使用这些指标

**在以下情况使用：**
- 评估是否扩大获客规模（LTV:CAC、回收期、魔术数字）
- 决定功能投资（利润率影响、对LTV的贡献）
- 规划跑道和融资（消耗率、跑道、40法则）
- 比较客户细分或渠道（按细分的单位经济）
- 董事会/投资者报告（40法则、魔术数字、LTV:CAC）
- 在增长与盈利能力之间做选择（40法则权衡）

**在以下情况不使用：**
- 在没有收入背景的情况下做决策（需结合 `saas-revenue-growth-metrics`）
- 在没有标准化的情况下比较差异巨大的业务模型
- 早期产品发现（收入前关注产品市场契合度，而非单位经济）
- 短期战术决策（使用参与度指标，而非LTV）

## 应用

### 第 1 步：计算单位经济指标

使用 `template.md` 中的模板计算你的单位经济指标。

#### 毛利率
```
毛利率 = (收入 - COGS) / 收入 × 100

COGS 包括：
- 托管与基础设施成本
- 支付处理费
- 客户入职成本
- 直接交付成本
```

**示例：**
- 收入：$1,000,000
- COGS：$200,000（托管$120K，处理$50K，入职$30K）
- 毛利率 = ($1M - $200K) / $1M = 80%

**质量检查：**
- 随着规模扩大，毛利率是否在提高？（应受益于规模经济）
- 哪些产品/功能利润率最高？（优先考虑这些）
- 利润率是否 >70%？（SaaS应是高利润率）

---

#### CAC（客户获取成本）
```
CAC = 总销售与营销支出 / 新获取客户数

销售与营销支出包括：
- 营销人员工资与工具
- 销售人员工资与佣金
- 广告与付费渠道
- SDR/BDR团队成本
```

**示例：**
- 销售与营销支出：$500,000/月
- 新客户：100/月
- CAC = $500,000 / 100 = $5,000

**质量检查：**
- 各渠道的CAC是否一致？（按渠道计算）
- CAC随时间推移是增加还是减少？（应随规模扩大而减少）
- CAC是否因客户细分而异？（SMB vs. 企业级）

---

#### LTV（生命周期价值）
```
LTV（简单） = ARPU × 平均客户生命周期（月）

LTV（更好） = ARPU × 毛利率 % / 月度流失率

LTV（高级） = 考虑扩展、特定群组留存率、折现率
```

**示例（简单）：**
- ARPU：$500/月
- 平均生命周期：36个月
- LTV = $500 × 36 = $18,000

**示例（更好）：**
- ARPU：$500/月
- 毛利率：80%
- 月度流失率：2%
- LTV = ($500 × 80%) / 2% = $400 / 0.02 = $20,000

**质量检查：**
- LTV是否随时间增长？（来自扩展、改善的留存率）
- LTV是否因群组而异？（新客户价值更高/更低？）
- LTV是否因细分市场而异？（企业级 vs. SMB）

---

#### LTV:CAC比率
```
LTV:CAC比率 = LTV / CAC
```

**示例：**
- LTV：$20,000
- CAC：$5,000
- LTV:CAC = $20,000 / $5,000 = 4:1

**质量检查：**
- 比率是否 >3:1？（可持续增长的最低要求）
- 比率是否 >5:1？（可能在增长上投入不足）
- 比率随时间推移是改善还是恶化？

**解读：**
- **<1:1** = 每个客户都在亏钱（不可持续）
- **1-3:1** = 边际经济（扩张前需优化）
- **3-5:1** = 健康（可放心扩张）
- **>5:1** = 可能投入不足（可以增长更快）

---

#### 回收期
```
回收期（月） = CAC / (月度ARPU × 毛利率 %)
```

**示例：**
- CAC：$5,000
- 月度ARPU：$500
- 毛利率：80%
- 回收期 = $5,000 / ($500 × 80%) = $5,000 / $400 = 12.5个月

**质量检查：**
- 回收期是否 <12个月？（优秀）

#### 贡献利润率
```
贡献利润率 = (收入 - 所有可变成本) / 收入 × 100

可变成本包括：
- 销货成本
- 支持成本（可变部分）
- 支付处理费
- 可变的客户成功成本
```

**示例：**
- 收入：$1,000,000
- 销货成本：$200,000
- 可变支持成本：$50,000
- 支付处理费：$30,000
- 贡献利润率 = ($1M - $280K) / $1M = 72%

**质量检查：**
- 贡献利润率是否 >60%？（对 SaaS 来说很好）
- 某些产品/细分市场的利润率是否较低？（考虑停止运营）
- 利润率是否随规模扩大而提高？

### 第 2 步：计算资本效率

#### 烧钱率
```
总烧钱率 = 每月总现金支出
净烧钱率 = 每月总现金支出 - 每月收入
```

**示例：**
- 每月支出：$800,000
- 每月收入：$400,000
- 总烧钱率：$800,000/月
- 净烧钱率：$400,000/月

**质量检查：**
- 净烧钱率是否随时间下降？（通往盈利的路径）
- 考虑到跑道，烧钱率是否可持续？
- 烧钱率相对于收入是多少？（烧钱倍数）

---

#### 跑道
```
跑道（月） = 现金余额 / 每月净烧钱率
```

**示例：**
- 现金余额：$6,000,000
- 净烧钱率：$400,000/月
- 跑道 = $6M / $400K = 15 个月

**质量检查：**
- 你的跑道是否 >12 个月？（健康）
- 你的跑道是否 <6 个月？（危机——立即融资或削减开支）
- 你能否在跑道结束前达到下一个里程碑？

**规则：** 在跑道剩余 6-9 个月时开始融资，而不是 3 个月。

---

#### 运营费用
```
运营费用 = 销售与营销 + 研发 + 行政管理

按收入百分比跟踪：
销售与营销占收入百分比
研发占收入百分比
行政管理占收入百分比
```

**示例：**
- 收入：$10M/年
- 销售与营销：$5M（收入的 50%）
- 研发：$3M（收入的 30%）
- 行政管理：$1M（收入的 10%）
- 总运营费用：$9M（收入的 90%）

**质量检查：**
- 运营费用类别的增长是否慢于收入？（运营杠杆）
- 销售与营销支出是否高效？（检查魔法数字）
- 行政管理是否 < 收入的 15%？（应保持低位）

---

#### 净收入（利润率）
```
净收入 = 收入 - 销货成本 - 运营费用
利润率 % = 净收入 / 收入 × 100
```

**示例：**
- 收入：$10M
- 销货成本：$2M
- 运营费用：$9M
- 净收入 = $10M - $2M - $9M = -$1M（亏损）
- 利润率 = -10%

**质量检查：**
- 利润率是否随时间改善？（通往盈利的路径）
- 按当前增长率，你何时能达到盈亏平衡？
- 你是否在将亏损投资于增长？（如果 LTV:CAC 健康则可接受）

---

### 第 3 步：计算效率比率

#### 40 法则
```
40 法则 = 收入增长率 % + 利润率 %
```

**示例 1（增长模式）：**
- 收入增长：同比 80%
- 利润率：-30%
- 40 法则 = 80% + (-30%) = 50 ✅ 健康

**示例 2（成熟期）：**
- 收入增长：同比 25%
- 利润率：20%
- 40 法则 = 25% + 20% = 45 ✅ 健康

**示例 3（问题）：**
- 收入增长：同比 30%
- 利润率：-35%
- 40 法则 = 30% + (-35%) = -5 ❌ 不健康

**质量检查：**
- 40 法则是否 >40？（健康平衡）
- 40 法则是否 >25？（可接受）
- 40 法则是否 <25？（在没有足够增长的情况下烧钱）

**权衡：**
- 早期阶段：最大化增长，接受亏损（60% 增长，-20% 利润率 = 40）
- 增长阶段：平衡（40% 增长，5% 利润率 = 45）
- 成熟期：优先考虑盈利（20% 增长，25% 利润率 = 45）

---

#### 魔法数字
```
魔法数字 = (本季度收入 - 上季度收入) × 4 / 上季度销售与营销支出
```

**示例：**
- Q2 收入：$2.5M
- Q1 收入：$2.0M
- Q1 销售与营销支出：$800K
- 魔法数字 = ($2.5M - $2.0M) × 4 / $800K = $2M / $800K = 2.5

**质量检查：**
- 魔法数字是否 >0.75？（高效——扩大销售与营销支出）
- 魔法数字是否在 0.5-0.75 之间？（可接受——在扩大规模前优化）
- 魔法数字是否 <0.5？（低效——在增加支出前修复 GTM）

**解读：**
- **>1.0** = 每投入 $1 销售与营销费用，你获得 $1+ 的新 ARR（优秀）
- **0.75-1.0** = 高效，可自信地扩大规模
- **0.5-0.75** = 边际，在扩大规模前优化
- **<0.5** = 低效，在投资更多之前修复

---

#### 运营杠杆
随时间跟踪，以查看你是否在高效地扩大规模。

**示例：**
| 季度 | 收入 | 同比增长 | 运营费用 | 同比增长 | 杠杆 |
|---------|---------|------------|------|------------|----------|
| Q1 2024 | $8M | - | $6M | - | - |
| Q2 2024 | $10M | 25% | $7M | 17% | 正向 ✅ |
| Q3 2024 | $12M | 20% | $9M | 29% | 负向 ⚠️ |

**质量检查：**
- 收入增长是否快于运营费用？（正向杠杆）
- 你是否相对于收入过快地扩大了运营费用？
- 哪个运营费用类别增长最快？（研发、销售与营销、行政管理）

---

### 第 4 步：按细分市场和渠道分析

**单位经济效益因细分市场而异：**

| 细分市场 | CAC | LTV | LTV:CAC | 回收期 | 毛利率 |
|---------|-----|-----|---------|---------|--------------|
| 中小企业 | $500 | $2,000 | 4:1 | 8 个月 | 75% |
| 中端市场 | $5,000 | $25,000 | 5:1 | 12 个月 | 80% |
| 大型企业 | $50,000 | $300,000 | 6:1 | 24 个月 | 85% |

**质量检查：**
- 哪个细分市场的单位经济效益最好？
- 哪个细分市场的回收期最快？（现金紧张时优先考虑）
- 哪个细分市场的 LTV 最高？（投资于留存/扩展）

## 示例

详细场景请参见 `examples/` 文件夹。以下是简要示例：

### 示例 1：健康的单位经济效益

**公司：** CloudAnalytics（中端市场分析 SaaS）

**单位经济效益：**
- CAC：$8,000
- LTV：$40,000
- LTV:CAC：5:1 ✅
- 回收期：10 个月 ✅
- 毛利率：82% ✅

**资本效率：**
- 每月净烧钱率：$300K
- 跑道：18 个月 ✅
- 40 法则：55（40% 增长 + 15% 利润率）✅
- 魔法数字：0.9 ✅

**分析：**
- 强劲的单位经济效益（5:1 LTV:CAC，10 个月回收期）
- 高效的 GTM（0.9 魔法数字）
- 健康的平衡（40 法则 = 55）
- 充足的跑道（18 个月）

**行动：** 积极扩大获客。经济模型支持增长。

---

### 示例 2：良好的 LTV:CAC，糟糕的回收期（现金陷阱）

**公司：** EnterpriseCRM（企业销售模式）

**单位经济效益：**
- CAC：$80,000
- LTV：$400,000
- LTV:CAC：5:1 ✅（看起来很棒！）
- 回收期：36 个月 ❌（糟糕！）
- 毛利率：85%

**资本效率：**
- 每月净烧钱率：$2M
- 跑道：9 个月 ❌
- 平均客户生命周期：48 个月
- 平均合同额：$100K/年

**分析：**
- ⚠️ 良好的 LTV:CAC 比率（5:1）掩盖了现金问题
- ❌ 36 个月的回收期配合 9 个月的跑道 = 现金陷阱
- ❌ 需要 3 年才能收回 CAC，但只有 9 个月的现金
- ⚠️ 客户留存 4 年，所以如果你有现金，经济模型是可行的

**问题：** 你将在收回获客成本之前耗尽现金。

**行动：**
1. 谈判预付年度付款（将回收期缩短至 12 个月）
2. 融资以延长跑道（需要 36+ 个月来维持增长）
3. 降低 CAC（缩短销售周期，提高转化率）
4. 瞄准回收期更快的小额交易（中端市场 vs. 大型企业）

---

### 示例 3：扩张过快（负运营杠杆）

**公司：** SocialScheduler（中小企业社交媒体工具）

**季度趋势：**
| 季度 | 收入 | 运营费用 | 净收入 | 收入增长 | 运营费用增长 |
|---------|---------|------|------------|----------------|-------------|
| Q1 | $1.0M | $800K | -$800K | - | - |
| Q2 | $1.3M | $1.2M | -$1.2M | 30% | 50% ❌ |
| Q3 | $1.6M | $1.8M | -$1.8M | 23% | 50% ❌ |

**分析：**
- ❌ 运营费用增长快于收入（50% vs. 23-30%）
- ❌ 亏损加速（2 个季度内从 $800K 增至 $1.8M）
- ❌ 负运营杠杆（应为正向）
- ⚠️ 在没有相应收入增长的情况下扩大了销售与营销和研发

**问题：** 在收入增长放缓的同时，现金消耗速度加快。

**行动：**
1. 冻结招聘，直到收入跟上
2. 削减低效的销售与营销支出（魔法数字可能 <0.5）
3. 在扩大规模前专注于改善单位经济效益
4. 目标是运营费用增长 < 收入增长

## 常见陷阱

### 陷阱 1：庆祝高 LTV 而不检查回收期
**症状：** “我们的 LTV:CAC 是 6:1，太棒了！”

**后果：** 6:1 的比率配合 48 个月的回收期是一个现金陷阱。你将在收回 CAC 之前耗尽资金。

**修复：** 始终将 LTV:CAC 与回收期配对。3:1 配合 10 个月回收期胜过 6:1 配合 36 个月回收期。

---

### 陷阱 2：计算 LTV 时忽略毛利率
**症状：** “LTV = $100/月 × 36 个月 = $3,600”

**后果：** 你使用的是收入，而不是利润。扣除 30% 销货成本后的实际 LTV 是 $2,520，而不是 $3,600。

**修复：** 在 LTV 计算中始终包含毛利率。`LTV = ARPU × 利润率 % / 流失率`。

---

### 陷阱 3：在魔法数字低时扩大销售与营销
**症状：** “我们需要增长更快——让我们把销售与营销支出翻倍！”（魔法数字 = 0.3）

**后果：** 你正在往一个坏掉的引擎里倒汽油。加倍支出只会加速现金消耗，而不会带来相应的收入增长。

**修复：** 仅在魔法数字 >0.75 时扩大销售与营销。如果 <0.5，先修复 GTM 效率。

---

### 陷阱 4：使用过于简单的 LTV 公式
**症状：** “LTV = ARPU × 生命周期”（忽略了扩展、折现率、队列差异）

**后果：** 高估了用于决策的 LTV。现实是：扩展提升 LTV；折扣降低 LTV；队列存在差异。

**修复：** 对重大决策使用复杂的 LTV 模型。简单的 LTV 仅适用于方向性指导。

---

### 陷阱 5：忘记货币的时间价值
**症状：** “今天的 $10K 收入 = 5 年后的 $10K 收入”

**后果：** 高估了回收期长的业务的 LTV。5 年后的 $10K 在今天价值约 $7.8K（按 5% 折现率）。

**修复：** 对于 LTV 周期 >24 个月的情况，对未来现金流进行折现。使用 NPV（净现值）。

---

### 陷阱 6：比较不同回收期的 CAC
**症状：** “渠道 A 的 CAC 是 $5K，渠道 B 的 CAC 是 $8K——渠道 A 更好！”

**后果：** 如果渠道 A 的回收期是 24 个月，渠道 B 的回收期是 8 个月，那么渠道 B 实际上更好（现金回收更快）。

**修复：** 将 CAC 和回收期一起比较，而不是孤立地看 CAC。

---

### 陷阱 7：在现金流为负时庆祝 40 法则 >40
**症状：** “40 法则 = 50，我们太棒了！”（60% 增长，-10% 利润率，每月烧钱 $5M）

**后果：** 40 法则没有考虑绝对烧钱量。你可能有很好的平衡，但只有 3 个月的跑道。

**修复：** 将 40 法则与烧钱率和跑道配对。平衡很重要，但生存更重要。

---

### 陷阱 8：忽略细分市场特定的单位经济效益
**症状：** “混合 CAC 是 $2K，混合 LTV 是 $10K，我们很好！”

**后果：** 中小企业细分市场可能有 $500 CAC / $2K LTV（很好），而大型企业有 $20K CAC / $15K LTV（糟糕）。混合指标掩盖了问题。

**修复：** 按细分市场计算单位经济效益。独立优化每个细分市场。

---

### 陷阱 9：混淆毛利率与贡献利润率
**症状：** “毛利率是 80%，我们的利润率很棒！”

**后果：** 扣除可变支持成本（10%）和支付处理费（3%）后，贡献利润率可能是 67%——而不是 80%。

**修复：** 同时跟踪毛利率（仅销货成本）和贡献利润率（所有可变成本）。在单位经济效益中使用贡献利润率。

---

### 陷阱 10：忘记营运资金的时间安排
**症状：** “根据烧钱率，我们有 12 个月的跑道”（但所有合同都是按月付款）

**后果：** 预付的年度合同会暂时增加现金。按月合同会延迟现金回收。跑道比烧钱率显示的更长/更短。

**修复：** 在计算跑道时考虑营运资金。基于现金的跑道 ≠ 基于收入的跑道。

# PoL 探测模板

使用此模板记录一次生命证明探测。

## 模板
```markdown
# PoL 探测：[描述性名称]

## 假设
[一句话说明你相信什么]
示例："如果我们把 onboarding 表单缩减到 3 个字段，完成率会超过 80%。"

## 消除的风险
[你在解决什么具体风险或未知？]
示例："我们不知道用户会不会因为表单太长而放弃注册。"

## 原型类型
[从 5 种类型中选一种]
- [ ] 可行性检查
- [ ] 任务聚焦测试
- [ ] 叙事原型
- [ ] 合成数据模拟
- [ ] Vibe-Coded PoL 探测

## 目标用户 / 受众
[谁会与这个探测交互？]
示例："10 位来自早期候补名单的用户，非技术型 SMB 经营者。"

## 成功标准（残酷真相）
[你想求证什么真相？什么结果能证明你错了？]
- **通过：** 8 位以上用户在 2 分钟内完成注册
- **失败：** 少于 6 位完成，或平均耗时超过 5 分钟
- **学习：** 找出具体流失字段

## 工具 / 技术栈
[你打算用什么搭建？]
示例："ChatGPT Canvas 搭建表单 UI，Airtable 收集数据，Loom 进行测试后访谈。"

## 时间线
- **构建：** 2 天
- **测试：** 1 天（10 场用户测试）
- **分析：** 1 天
- **处置：** 第 5 天（删除全部代码，只保留学习文档）

## 处置计划
[什么时候删？怎么删？]
示例："用户测试结束后，归档录像，删除 Frankensoft 代码，把学习结论写进 Notion。"

## 负责人
[谁对执行和销毁负责？]

## 状态
- [ ] 假设已定义
- [ ] 探测已构建
- [ ] 用户已招募
- [ ] 测试已完成
- [ ] 学习已记录
- [ ] 探测已处置
```

# Jobs-to-be-Done 模板

使用此模板以结构化方式记录客户任务、痛点和收益。

## 来源
改编自 `https://github.com/deanpeters/product-manager-prompts` 仓库中的 `prompts/jobs-to-be-done.md`。

## 模板
```markdown
## Jobs-to-be-Done

### 1. 客户任务

#### 功能性任务：
- [功能性任务 1]
- [功能性任务 2]
- [功能性任务 3]

#### 社会性任务：
- [客户期望的形象 1]
- [客户期望的形象 2]
- [客户期望的形象 3]

#### 情感性任务：
- [希望达成或避免的情绪状态 1]
- [希望达成或避免的情绪状态 2]
- [希望达成或避免的情绪状态 3]

### 2. 痛点

#### 挑战：
- [障碍 1]
- [障碍 2]
- [障碍 3]

#### 高成本：
- [时间、金钱或精力上的高成本 1]
- [时间、金钱或精力上的高成本 2]

#### 常见错误：
- [常见错误 1]
- [常见错误 2]

#### 未解决的问题：
- [当前方案未解决的问题 1]
- [当前方案未解决的问题 2]

### 3. 收益

#### 期望：
- [超出预期的点 1]
- [超出预期的点 2]

#### 节省：
- [时间/金钱/精力节省 1]
- [时间/金钱/精力节省 2]

#### 采用因素：
- [提高采用可能性的因素 1]
- [提高采用可能性的因素 2]

#### 生活改善：
- [生活如何变好 1]
- [生活如何变好 2]
```

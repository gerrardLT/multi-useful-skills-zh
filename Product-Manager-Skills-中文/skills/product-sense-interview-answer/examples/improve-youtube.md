# 示例：改进 YouTube

**Prompt:** `How would you improve YouTube?`

## 为什么这是一个优秀示例

这个例子好在，它没有把 “YouTube” 当成一个巨大而无差别的整体产品。
它先收窄目标用户，再挑出一个痛点，最后通过显式权衡才得出最小可行产品选择。

## 精简步骤解析

### 1. 澄清

- 把范围限定在核心的 YouTube 观看体验，而不是 Shorts、Music 或 TV
- 澄清关注对象是观众、创作者还是广告主
- 如果面试官不回答，就默认整个生态都在范围内，但明确会优先聚焦一个参与者

### 2. 原理

- 在线视频是一个巨大的注意力市场，而且竞争非常激烈
- 更深层的问题不只是观看时长，而是用户是否觉得自己的时间花得值
- YouTube 的独特优势在于创作者供给的广度，覆盖学习、娱乐和各种细分内容
- 核心论点：推荐系统更偏向优化参与度，而不是有意识的满足感

### 3. 产品目标

帮助观众稳定找到那些他们看完后会庆幸自己看过的内容，从而让 YouTube 成为一个用户“有意识地选择”而不是“习惯性地刷”的平台。

### 4. 市场细分

**生态系统参与者：**

- 观众
- 创作者
- 广告主
- 人才经纪人 / 机构
- 审核与信任团队

**选定参与者：** 观众

**主要维度：** 观看意图

- 目标导向的学习
- 娱乐性浏览
- 深度研究

**选定细分市场：** 目标导向的学习者

**次要维度：** 专业水平

- 初学者
- 中级
- 高级

**选定细分市场：** 初学者

**用户画像：** Priya 是一名 28 岁的市场协调员，她用 YouTube 快速学习实用工作技能。她在意的是：能否找到值得信赖、结构清晰的内容，同时不被低信号的推荐循环浪费时间。

### 5. 痛点

**旅程阶段：**

- 搜索
- 选择
- 会话内学习
- 后续跟进

**关键痛点：**

- 对一个主题没有清晰的起点
- 点开前很难判断质量
- 太多教程看起来都差不多
- 旧内容会压过更新更好的内容
- 推荐系统会把用户从原任务中带偏
- 从一个概念到下一个概念之间没有结构化的进阶路径
- 没有学习进度记录

**首要痛点：** 缺乏结构化的进阶路径

- **频率：** 几乎每次学习型会话在看完第一个视频后都会出现
- **严重性：** 它直接阻塞了用户要完成的任务，因为技能学习需要“顺序”，而不是一堆随机相邻内容

### 6. 解决方案

**选项 1：** 学习路径，用一条引导式课程把最好的初学者视频串起来

**选项 2：** 意图感知推荐，让整个会话尽量围绕用户声明的学习目标展开

**选项 3：** 关键时刻索引，让用户直接跳进视频内部自己最需要的概念片段

**比较：**

- 学习路径：**高用户影响**，直接解决进阶路径问题；**高实现成本**，因为涉及排序、编排和新用户界面
- 意图感知推荐：**中等用户影响**，能减少分心；**中等实现成本**，因为要改推荐系统
- 关键时刻索引：**中等用户影响**，提升视频内导航效率；**中等实现成本**，因为涉及转录文本和章节标注

**最小可行产品胜出者：** 学习路径

**核心功能：**

- 按主题组织的 5-8 个视频学习路径
- 跨会话进度追踪
- `继续学习` 的回流入口

**v1 排除项：**

- 不做测验
- 不做创作者自提交路径编辑

**结束语：**

`我会聚焦于目标导向的初学者学习者，特别是针对缺乏结构化进阶路径的痛点，并将学习路径作为第一个赌注。`

## 这个例子里值得注意的点

- 目标说的是结果，不是功能
- 细分市场足够窄，因而真正有意义
- 痛点是真实用户摩擦，而不是“少一个功能”
- 最终最小可行产品是做完比较后选出来的，不是凭直觉先拍脑袋定出来的

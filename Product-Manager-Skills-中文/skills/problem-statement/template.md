# 问题陈述模板

使用此模板从用户画像的视角界定用户问题，记录背景与约束，并产出一句话的问题陈述。

## 来源
改编自 `https://github.com/deanpeters/product-manager-prompts` 仓库中的 `prompts/framing-the-problem-statement.md`。

## 模板
```markdown
## 问题框架叙述

**我是：** [描述关键用户画像，突出 3-4 个核心特征]
- [关键痛点或特征 1]
- [关键痛点或特征 2]
- [关键痛点或特征 3]

**试图：**
- [用一句话列出用户画像最关心的期望结果]

**但是：**
- [描述阻碍结果实现的障碍]
- [结果阻碍 1]
- [结果阻碍 2]
- [结果阻碍 3]

**因为：**
- [以共情的方式描述根本原因]

**这让我感觉：**
- [从用户画像的视角描述情绪]

## 背景与约束
- [地理、技术、时间或人口统计因素]

## 最终问题陈述
- [一句简洁、共情的总结性陈述]
```

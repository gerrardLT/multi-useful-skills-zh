# 用户故事拆分示例

## 示例 1：按工作流步骤拆分

**原始用户故事：**
```markdown
### 用户故事 100：完成结账流程

**用例：**
- **作为** 一名购物者
- **我想要** 完成结账，包括输入配送信息、支付信息并确认订单
- **以便** 我能收到购买的商品

**验收标准：**
- **假设：** 我的购物车中有商品
- **当：** 我输入配送地址、支付信息并确认
- **那么：** 我的订单被创建
```

**为什么需要拆分：** 将多个工作流步骤打包在了一起。

**按工作流步骤拆分：**

```markdown
### 拆分 1：输入配送地址

**用户故事 101：**
- **概要：** 在结账过程中输入配送地址

**用例：**
- **作为** 一名购物者
- **我想要** 输入我的配送地址
- **以便** 我的商品能被送到正确的位置

**验收标准：**
- **假设：** 我的购物车中有商品
- **当：** 我输入一个有效的配送地址并点击“继续”
- **那么：** 我进入支付信息输入步骤

---

### 拆分 2：输入支付信息

**用户故事 102：**
- **概要：** 在结账过程中输入支付信息

**用例：**
- **作为** 一名购物者
- **我想要** 输入我的信用卡信息
- **以便** 我能支付订单

**验收标准：**
- **假设：** 我已输入配送地址
- **当：** 我输入有效的支付信息并点击“继续”
- **那么：** 我进入订单确认步骤

---

### 拆分 3：确认订单

**用户故事 103：**
- **概要：** 查看并确认订单

**用例：**
- **作为** 一名购物者
- **我想要** 查看我的订单详情并确认
- **以便** 我的订单被创建

**验收标准：**
- **假设：** 我已输入配送和支付信息
- **当：** 我查看订单摘要并点击“下单”
- **那么：** 我的订单被提交，并且我收到确认信息
```

---

## 示例 2：按验收标准复杂度拆分

**原始用户故事：**
```markdown
### 用户故事 200：管理团队成员

**用例：**
- **作为** 一名团队管理员
- **我想要** 管理团队成员
- **以便** 我能控制访问权限

**验收标准：**
- **当：** 我添加一名成员，那么他们会收到邀请
- **当：** 我移除一名成员，那么他们会失去访问权限
- **当：** 我更改一名成员的角色，那么他们的权限会更新
```

**为什么需要拆分：** 同时包含多组 `When/Then`。

**按验收标准复杂度拆分：**

```markdown
### 拆分 1：添加团队成员

**用户故事 201：**
- **概要：** 邀请新的团队成员

**用例：**
- **作为** 一名团队管理员
- **我想要** 邀请新成员加入我的团队
- **以便** 他们可以访问共享资源

**验收标准：**
- **假设：** 我拥有管理员权限
- **当：** 我输入一个邮箱地址并点击“邀请”
- **那么：** 收件人会收到一封邀请邮件

---

### 拆分 2：移除团队成员

**用户故事 202：**
- **概要：** 移除团队成员

**用例：**
- **作为** 一名团队管理员
- **我想要** 从我的团队中移除成员
- **以便** 他们不再拥有访问权限

**验收标准：**
- **假设：** 我拥有管理员权限
- **当：** 我点击成员姓名旁的“移除”
- **那么：** 该成员立即失去访问权限

---

### 拆分 3：更改成员角色

**用户故事 203：**
- **概要：** 更新团队成员角色

**用例：**
- **作为** 一名团队管理员
- **我想要** 更改一名成员的角色
- **以便** 他们的权限与其职责相匹配

**验收标准：**
- **假设：** 我拥有管理员权限
- **当：** 我为一名成员选择一个新角色并保存
- **那么：** 他们的权限更新以匹配新角色
```

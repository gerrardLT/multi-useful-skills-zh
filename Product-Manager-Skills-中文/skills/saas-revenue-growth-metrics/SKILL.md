## 目的

掌握收入和留存指标，以理解 SaaS 业务动能，评估产品市场契合度，并基于数据做出关于增长投资的决策。用它来计算关键指标、解读趋势、早期发现问题，并向利益相关者传达业务健康状况。

这不是商业智能工具，而是一个给 PM 用的分析框架，帮助你理解哪些指标最关键、该如何正确计算，以及看完数字后应该采取什么行动。

## 关键概念

### 收入指标家族

衡量业务产生多少收入的“顶线”指标。

**收入** — 销售产品/服务所获得的总金额，未扣除费用。损益表的“顶线”。
- **为什么 PM 关心：** 每个功能都应与收入（直接或间接）挂钩。如果你无法阐述收入影响，优先级排序就无从谈起。
- **公式：** 一个时期内所有客户付款的总和
- **基准：** 增长率比绝对数字更重要（因阶段和上下文而异）

**ARPU（每用户平均收入）** — 每个独立用户产生的平均收入。
- **为什么 PM 关心：** 衡量每席位的变现效率。对于基于席位的定价模型至关重要。
- **公式：** `总收入 / 总用户数`
- **基准：** 因模型而异；跟踪趋势比绝对值更重要
- **B2C SaaS：** 典型值 $5-50/月；B2B：$50-500+/月

**ARPA（每账户平均收入）** — 每个客户账户产生的平均收入。
- **为什么 PM 关心：** 衡量账户级别的交易规模。对于基于账户的定价模型至关重要。
- **公式：** `MRR / 活跃账户数`
- **基准：** SMB SaaS：$100-$1K/月；中型市场：$1K-$10K；企业级：$10K+

**ARPA/ARPU 分析** — 结合使用两个指标来理解变现情况。
- **为什么 PM 关心：** 防止打包错误。高 ARPA + 低 ARPU = 每席位变现不足。低 ARPA + 高 ARPU = 交易规模小。
- **示例：** $10K ARPA，100 个席位 = $100 ARPU（合理）。$10K ARPA，1,000 个席位 = $10 ARPU（钱没赚够）。

**ACV（年度合同价值）** — 每份合同的年度化经常性收入（不包括一次性费用）。
- **为什么 PM 关心：** 比较不同合同结构的经济性。有助于销售薪酬设计和细分市场分析。
- **公式：** `每份合同的年度经常性收入`（不包括设置费、专业服务费）
- **基准：** SMB：$5K-$25K；中型市场：$25K-$100K；企业级：$100K+

**MRR/ARR（月度/年度经常性收入）** — 标准化为月度或年度的可预测经常性收入。
- **为什么 PM 关心：** 订阅业务的命脉。估值倍数可达 5-10 倍以上。跟踪其组成部分（新增、扩展、流失）。
- **公式：** `MRR = 每月所有经常性订阅收入总和`；`ARR = MRR × 12`
- **基准：** 增长率和质量很重要；跟踪新增 MRR、扩展 MRR、流失 MRR、收缩 MRR

**总收入 vs. 净收入** — 折扣、退款、信用额度调整前的总收入 vs. 调整后的净收入。
- **为什么 PM 关心：** 折扣和退款可能掩盖糟糕的获客质量或产品问题。
- **公式：** `净收入 = 总收入 - 折扣 - 退款 - 信用额度`
- **基准：** 退款率 >10% 是危险信号；按获客渠道跟踪

---

### 留存与扩展指标家族

衡量你留住并增长现有客户能力的指标。

**流失率** — 一个时期内取消服务的客户百分比。
- **为什么 PM 关心：** SaaS 的无声杀手。破坏所有获客努力。5% 的月流失率 = 46% 的年流失率（复利效应）。
- **公式：** `期间流失客户数 / 期初客户数`
- **基准（月度）：** <2% 优秀，2-5% 可接受，>5% 危机
- **基准（年度）：** <10% 优秀，10-30% 可接受，>30% 危机
- **注意：** Logo 流失（客户数量）与收入流失（金额）不同

**NRR（净收入留存）** — 来自现有客户的收入留存，包括扩展和收缩。
- **为什么 PM 关心：** 圣杯指标。NRR >100% 意味着无需新客户也能增长。投资者高度看重。
- **公式：** `(期初 ARR + 扩展 - 流失 - 收缩) / 期初 ARR × 100`
- **基准：** >120% 优秀，100-120% 良好，90-100% 可接受，<90% 有问题
- **示例：** 期初 ARR $1M，新增扩展 $300K，流失 $100K = $1.2M / $1M = 120% NRR

**扩展收入** — 来自现有客户的额外收入（追加销售、交叉销售、使用量增长）。
- **为什么 PM 关心：** 资本效率最高的收入（无获客成本）。应驱动 NRR >100%。
- **公式：** `来自现有客户的追加销售 + 交叉销售 + 使用量增长总和`
- **基准：** 应占总收入的 20-30%；驱动 NRR >100%

**速动比率（SaaS）** — 收入增长 vs. 收入损失。
- **为什么 PM 关心：** 显示你是在坚实的基础上建设，还是在跑步机上奔跑。
- **公式：** `(新增 MRR + 扩展 MRR) / (流失 MRR + 收缩 MRR)`
- **基准：** >4 优秀，2-4 健康，<2 漏桶

---

### 分析框架

**收入构成分析** — 按产品、细分市场或渠道分解收入。
- **为什么 PM 关心：** 识别哪些产品为业务提供资金以及投资方向。揭示集中度风险。
- **公式：** `产品/细分市场收入 / 总收入 × 100`
- **基准：** 理想情况下没有单一产品占比 >60%；多元化降低风险

**队列分析** — 按加入日期对客户分组，并随时间跟踪其行为。
- **为什么 PM 关心：** 混合指标会掩盖关键趋势。显示业务是在改善还是在恶化。
- **方法：** 按队列（例如“2024年1月队列”）跟踪留存、扩展和 LTV
- **基准：** 近期队列应表现与旧队列相同或更好

---

### 反模式（它不是什么）

- **不是利润指标：** 收入是顶线，不是底线。高收入但负利润率是灾难。
- **不是虚荣指标：** 如果由不可持续的折扣或破坏利润的交易驱动，总收入增长毫无意义。
- **不是混合平均值：** 将 $10 的 SMB 和 $1,000 的企业客户平均得出的 ARPU 隐藏了细分市场的经济性。
- **不是孤立数字：** 单独的流失率无法说明问题——需要查看队列趋势和 NRR。

---

### 何时使用这些指标

**在以下情况使用：**
- 评估整体业务健康状况和产品市场契合度
- 比较不同时间段或队列的表现
- 优先考虑具有直接变现路径的功能（ARPU 影响、扩展推动因素）
- 与领导层、董事会或投资者沟通
- 评估留存问题（流失分析、队列恶化）
- 衡量定价或打包变更（ARPU/ARPA 变化）

**在以下情况不使用：**
- 评估盈利能力（改用利润率指标）
- 评估资本效率（改用 LTV:CAC、回收期）
- 在没有成本背景的情况下做出产品投资决策（收入本身不是 ROI）
- 在未标准化的情况下比较截然不同的商业模式

## 应用

### 第 1 步：计算收入指标

使用 `template.md` 中的模板计算你的核心收入指标。

#### 收入
```
收入 = 期间所有客户付款总和
```

**示例：**
- 第 1 个月付款：$100,000
- 收入 = $100,000

**质量检查：**
- 这是总收入还是净收入？（明确是否包含折扣/退款）
- 收入增长是来自队列同比，还是仅来自新客户增加？
- 收入增长率与人员/成本增长率相比如何？

---

#### ARPU（每用户平均收入）
```
ARPU = 总收入 / 总用户数
```

**示例：**
- 总收入：$100,000/月
- 总用户数：2,000
- ARPU = $100,000 / 2,000 = $50/用户/月

**质量检查：**
- ARPU 随时间是在增长还是萎缩？
- ARPU 增长是来自价格上涨还是组合变化（失去小客户）？
- ARPU 如何因队列而异？（新客户价值是否较低？）

---

#### ARPA（每账户平均收入）
```
ARPA = MRR / 活跃账户数
```

**示例：**
- MRR：$100,000
- 活跃账户数：200
- ARPA = $100,000 / 200 = $500/账户/月

**质量检查：**
- ARPA 增长是来自扩展还是仅来自更大的新交易？
- ARPA 在不同客户细分市场间如何比较？
- ARPA 高但 ARPU 低？（每席位变现不足）

---

#### ARPA/ARPU 组合分析
```
ARPA = MRR / 活跃账户数
ARPU = MRR / 总用户数
每账户平均席位数 = ARPA / ARPU
```

**示例：**
- ARPA：$500/月
- ARPU：$50/月
- 平均席位数：$500 / $50 = 10 席位/账户

**质量检查：**
- 你是否有效地按席位变现？
- 你能提高每席位收费（提高 ARPU）吗？
- 你能扩展每账户的席位数（提高 ARPA）吗？

---

#### ACV（年度合同价值）
```
ACV = 每份合同的年度经常性收入
（不包括一次性费用，如设置费、专业服务费）
```

**示例：**
- 客户签订 3 年合同，总金额 $300K
- ACV = $300K / 3 年 = $100K/年

**质量检查：**
- ACV 如何因细分市场（SMB vs. 企业级）而异？
- ACV 是否随时间增长（向高端市场移动）？
- ACV 是否证明销售团队成本结构合理？

---

#### MRR/ARR（月度/年度经常性收入）
```
MRR = 所有月度经常性订阅总和
ARR = MRR × 12

跟踪组成部分：
- 新增 MRR（来自新客户）
- 扩展 MRR（来自追加销售/交叉销售）
- 流失 MRR（来自流失客户）
- 收缩 MRR（来自降级）
```

**示例：**
- 期初 MRR：$500K
- 新增 MRR：+$50K
- 扩展 MRR：+$20K
- 流失 MRR：-$15K
- 收缩 MRR：-$5K
- 期末 MRR：$550K
- ARR = $550K × 12 = $6.6M

**质量检查：**
- MRR 增长是来自新客户还是扩展？
- 流失/收缩是否随着增长而增加？
- 新增:扩展:流失 MRR 的比率是多少？（最佳：扩展 > 新增）

---

#### 总收入 vs. 净收入
```
净收入 = 总收入 - 折扣 - 退款 - 信用额度
```

**示例：**
- 总收入：$100K
- 折扣：-$10K
- 退款：-$2K
- 净收入：$88K

**质量检查：**
- 折扣率 >20%？（定价权问题）
- 退款率 >10%？（产品质量问题）
- 某些渠道是否有更高的折扣/退款率？

---

### 第 2 步：计算留存与扩展指标

#### 流失率
```
Logo 流失率 = 流失客户数 / 期初客户数 × 100
收入流失率 = 流失 MRR / 期初 MRR × 100
```

**示例（Logo 流失）：**
- 期初客户数：1,000
- 流失客户数：30
- Logo 流失率 = 30 / 1,000 = 3% 月度

**示例（收入流失）：**
- 期初 MRR：$500K
- 流失 MRR：$15K
- 收入流失率 = $15K / $500K = 3% 月度

**质量检查：**
- 流失率随时间是在加速还是减速？
- 较新的队列是否比较旧的队列流失更快？（PMF 退化）
- 收入流失率是否高于 Logo 流失率？（失去大客户）

**将月度转换为年度：**
- 月度流失复利：3% 月度 ≈ 36% 年度
- 公式：`年度流失率 = 1 - (1 - 月度流失率)^12`
- 3% 月度 = ~31% 年度流失率

---

#### NRR（净收入留存）
```
NRR = (期初 ARR + 扩展 - 流失 - 收缩) / 期初 ARR × 100
```

**示例：**
- 期初 ARR：$5M
- 扩展：+$800K
- 流失：-$300K
- 收缩：-$100K
- 来自该队列的期末 ARR：$5.4M
- NRR = $5.4M / $5M = 108%

**质量检查：**
- NRR 是否 >100%？（无需新客户也能增长）
- NRR 随队列同比是在改善还是恶化？
- 是什么驱动了 NRR？（扩展还是低流失？）

---#### 扩展收入
```
扩展收入 = 追加销售 + 交叉销售 + 使用量增长（来自现有客户）
```

**示例：**
- 追加销售至更高层级：每月 $50K
- 交叉销售附加组件：每月 $20K
- 使用量增长：每月 $10K
- 总扩展收入：每月 $80K

**质量检查：**
- 扩展收入占总收入的百分比是否在增长？
- 每年有多少百分比的客户进行扩展？（扩展率）
- 某些客户群/细分市场是否更有可能进行扩展？

---

#### 快速比率（SaaS）
```
快速比率 = （新增 MRR + 扩展 MRR）/ （流失 MRR + 收缩 MRR）
```

**示例：**
- 新增 MRR：$50K
- 扩展 MRR：$20K
- 流失 MRR：$15K
- 收缩 MRR：$5K
- 快速比率 = ($50K + $20K) / ($15K + $5K) = $70K / $20K = 3.5

**质量检查：**
- 快速比率 >4 = 优秀（收益远超损失）
- 快速比率 2-4 = 健康（可持续增长）
- 快速比率 <2 = 漏桶（在扩张前先解决留存问题）

---

### 第 3 步：使用框架分析趋势

#### 收入构成分析
```
产品/细分市场占比 = 产品/细分市场收入 / 总收入 × 100
```

**示例：**
- 产品 A 收入：$300K
- 产品 B 收入：$500K
- 产品 C 收入：$200K
- 总收入：$1M
- 产品 A：30%，产品 B：50%，产品 C：20%

**质量检查：**
- 收入集中度是否在增加？（风险：过度依赖单一产品）
- 哪些产品在增长/萎缩？
- 收入构成是否符合您的战略重点？

---

#### 群组分析
按客户加入时间分组，并跟踪其指标随时间的变化。

**示例：**
| 群组 | 第 0 月 | 第 1 月 | 第 2 月 | 第 3 月 | 第 6 月 |
|--------|---------|---------|---------|---------|---------|
| 2024年1月 | 100% | 95% | 92% | 90% | 85% |
| 2024年2月 | 100% | 94% | 90% | 87% | 80% |
| 2024年3月 | 100% | 92% | 86% | 82% | - |

**质量检查：**
- 近期群组的留存率比旧群组更好还是更差？
- 如果更差：产品市场契合度正在下降（在扩张前修复）
- 如果更好：改进措施正在起作用（可以安全扩张）
- 按群组跟踪收入留存率，而不仅仅是客户数量留存率

---

### 第 4 步：质量检查与基准

在报告指标之前，请验证：

**收入指标：**
- 鉁?总收入与净收入已明确标注
- 鉁?收入增长率 > 成本增长率
- 鉁?按群组分析了 ARPU/ARPA 趋势（而非仅看混合数据）

**留存指标：**
- 鉁?同时跟踪了客户数量流失和收入流失
- 鉁?分析了群组间的趋势（而非仅看混合流失率）
- 鉁?跟踪了 NRR 及其组成部分（扩展、流失、收缩）

**分析：**
- 鉁?群组分析显示了留存趋势
- 鉁?收入构成显示了集中风险
- 鉁?快速比率显示了增长可持续性

## 示例

详细场景请参见 `examples/` 文件夹。以下是简要示例：

### 示例 1：健康的 SaaS 指标

**公司：** 中端市场项目管理 SaaS

**收入指标：**
- MRR：$2M（月环比增长 10%）
- ARR：$24M
- ARPA：每月 $1,200（200 个账户）
- ARPU：每月 $120（20,000 名用户）
- 平均席位：每账户 100 个

**留存指标：**
- 月度客户数量流失率：2%
- 收入流失率：1.5%（流失的是较小客户）
- NRR：115%（强劲的扩展）
- 扩展收入：每月 $200K（占 MRR 的 10%）
- 快速比率：5.0

**分析：**
- 鉁?增长强劲（MRR 月环比 10%）
- 鉁?留存优秀（客户数量流失率 2%，NRR 115%）
- 鉁?扩展健康（NRR >100%）
- 鉁?可持续（快速比率 5.0）
- 鉁?收入流失率 < 客户数量流失率（流失的是较小客户，好信号）

**行动建议：** 扩大获客规模。单位经济效益强劲。

---

### 示例 2：警示信号

**公司：** 中小企业营销自动化 SaaS

**收入指标：**
- MRR：$500K（月环比增长 15%）
- ARR：$6M
- ARPA：每月 $250（2,000 个账户）
- ARPU：每月 $50（10,000 名用户）

**留存指标：**
- 月度客户数量流失率：6%（较六个月前的 4% 有所上升）
- 收入流失率：7%（流失的是较大客户）
- NRR：85%（正在收缩）
- 扩展收入：每月 $5K（占 MRR 的 1%）
- 快速比率：1.2

**群组分析：**
| 群组 | 第 6 月留存率 |
|--------|-------------------|
| 6个月前 | 75% |
| 3个月前 | 65% |
| 当前 | 58% |

**分析：**
- 鈿狅笍 高流失率（月度 6% ≈ 年度 ~50%）
- 馃毃 收入流失率 > 客户数量流失率（流失的是较大客户）
- 馃毃 NRR <100%（正在收缩，而非扩展）
- 馃毃 群组表现恶化（新客户流失更快）
- 馃毃 快速比率 1.2（漏桶）

**行动建议：** 停止扩大获客规模。首先解决留存问题。调查：
- 为什么新群组流失更快？
- 为什么扩展收入仅占 MRR 的 1%？
- 是什么导致了客户收缩？

---

### 示例 3：混合指标掩盖问题

**公司：** 多产品 SaaS 平台

**混合指标看起来很好：**
- MRR：$3M（月环比增长 20%）
- 混合流失率：3%
- 混合 NRR：110%

**但收入构成分析显示：**
| 产品 | 收入 | 占比 | 增长 | 流失率 | NRR |
|---------|---------|------------|--------|-------|-----|
| 旧产品 | $2M | 67% | 月环比 -5% | 8% | 75% |
| 新产品 | $1M | 33% | 月环比 +80% | 1% | 150% |

**分析：**
- 馃毃 旧产品（占收入 67%）正在衰退：增长 -5%，流失率 8%，NRR 75%
- 鉁?新产品表现出色：增长 +80%，流失率 1%，NRR 150%
- 鈿狅笍 混合指标掩盖了 2/3 收入正在收缩的事实
- 鈿狅笍 高度依赖单一产品（67% 的集中风险）

**行动建议：** 加速从旧产品向新产品的迁移。规划旧产品的日落计划。

## 常见陷阱

### 陷阱 1：混淆收入与利润
**症状：** “我们今年收入增长了 50%，我们太棒了！”

**后果：** 收入是顶线，不是底线。您可能在亏损中增长，破坏利润率，或在扩张不盈利的产品。

**解决方法：** 始终将收入指标与利润率指标配对（参见 `saas-economics-efficiency-metrics`）。80% 利润率的 $1M 收入 >> 20% 利润率的 $2M 收入。

---

### 陷阱 2：因结构变化而庆祝 ARPU 增长
**症状：** “ARPU 增长了 30%！”（但客户数量下降了 40%）

**后果：** ARPU 上升是因为您失去了所有小客户，而不是因为您提高了变现能力。

**解决方法：** 按群组和细分市场分析 ARPU。真正的 ARPU 提升 = 同样的客户支付更多，而不是失去廉价客户。

---

### 陷阱 3：忽视群组表现恶化
**症状：** “混合流失率稳定在 3%”

**后果：** 混合指标可能掩盖了新群组流失率为 6% 而旧群组流失率为 1% 的事实。产品市场契合度正在下降。

**解决方法：** 始终按群组分析留存率。如果新群组表现更差，请停止扩张并修复产品。

---

### 陷阱 4：混淆客户数量流失与收入流失
**症状：** “客户数量流失率只有 2%，我们很好！”

**后果：** 您可能流失了 2% 的客户，但如果流失的是大客户，收入可能流失了 10%。

**解决方法：** 同时跟踪客户数量流失率和收入流失率。如果收入流失率 > 客户数量流失率，说明您正在流失高价值客户。

---

### 陷阱 5：对所有流失一视同仁
**症状：** “我们这个月失去了 50 个客户”（没有关于客户是谁的背景信息）

**后果：** 失去 50 个小客户（每月 $10）与失去 50 个企业客户（每月 $10K）是完全不同的。

**解决方法：** 按客户规模、群组和原因细分流失分析。按收入影响加权，而不仅仅是客户数量。

---

### 陷阱 6：忘记复利流失
**症状：** “月度流失率 3% 没问题，年度也只有 36%”

**后果：** 流失是复利的。月度 3% = 年度 31% 流失率，而不是 36%。计算公式：`1 - (1 - 0.03)^12 = 31%`。

**解决方法：** 在将月度流失率转换为年度流失率时使用正确的公式。不要简单地乘以 12。

---

### 陷阱 7：在净收入收缩时庆祝总收入
**症状：** “总收入增长了 20%！”（但折扣/退款翻了一番）

**后果：** 净收入可能持平或萎缩。折扣掩盖了定价能力问题；退款掩盖了产品质量问题。

**解决方法：** 始终跟踪总收入和净收入。如果折扣 >20% 或退款 >10%，请调查原因。

---

### 陷阱 8：NRR >100% 来自低流失而非扩展
**症状：** “NRR 是 105%，我们在扩展！”

**后果：** NRR 可能仅因极低的流失率而 >100%，而没有实质性的扩展。真正的扩展驱动型 NRR 应 >120%。

**解决方法：** 将 NRR 分解为组成部分：扩展 MRR 与流失/收缩 MRR。目标是扩展驱动的 NRR，而不仅仅是低流失。

---

### 陷阱 9：收入集中风险
**症状：** “我们达到了 $10M ARR！”（但 $5M 来自一个客户）

**后果：** 失去那一个客户会使收入减半。产品路线图会受制于一个客户的要求。

**解决方法：** 跟踪收入集中度。理想情况：最大客户收入占比 <10%，前 10 大客户收入占比 <40%。尽早实现多元化。

---

### 陷阱 10：跨细分市场平均 ARPU/ARPA
**症状：** “我们的 ARPU 是 $100”（$10 的中小企业客户和 $1,000 的企业客户的平均值）

**后果：** 混合 ARPU 掩盖了细分市场的经济效益。无法做出明智的获客或产品决策。

**解决方法：** 按细分市场（中小企业、中端市场、企业）计算 ARPU/ARPA。独立优化每个细分市场。

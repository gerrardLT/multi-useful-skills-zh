# Finance Suite Summary

**SaaS 财务与单位经济学的 7 项技能**

**发布日期：** 2026年2月8日
**套件版本：** 1.0 (repo v0.3)
**技能总数：** 7 (3 个组件 + 4 个交互式)
**总行数：** 4,290
**覆盖范围：** 32 项 SaaS 财务指标

---

## 概述

Finance Suite 为产品经理提供了综合性工具，用于理解、计算和应用 SaaS 财务指标，以支持更明智的决策。该套件涵盖收入与增长指标、单位经济学、资本效率，并提供交互式顾问，用于功能优先级排序、渠道评估、定价决策和业务健康诊断。

**包含内容：**
- **基础层：** 深入理解指标（涵盖收入、经济学、效率的 32 项指标）
- **参考层：** 快速查询公式和基准
- **决策层：** 用于功能、渠道和定价决策的交互式顾问
- **诊断层：** 全面的业务健康评估

**未包含内容（未来第 8 阶段）：**
- 全面的定价策略（基于价值的定价、WTP 研究）
- 包装架构设计
- 定价心理学和行为策略
- 变现模式选择框架

## 技能详情

### 技能 #1：`saas-revenue-growth-metrics`（组件）

**目的：** 掌握收入和留存指标，以理解 SaaS 业务的增长势头。

**覆盖范围（13 项指标）：**
- 收入
- ARPU（每用户平均收入）
- ARPA（每账户平均收入）
- ARPA/ARPU 分析
- ACV（年度合同价值）
- MRR/ARR（月度/年度经常性收入）
- 总收入 vs. 净收入
- 流失率
- NRR（净收入留存率）
- 扩展收入
- 速动比率（SaaS）
- 收入构成分析
- 队列分析

**关键概念：**
- 收入指标家族
- 留存与扩展指标家族
- 分析框架（队列、收入构成）
- 反模式（虚荣指标、客户数流失 vs. 收入流失）

**支持文件：**
- `template.md` — 所有指标的填写计算器
- `examples/healthy-saas.md` — 健康指标示例
- `examples/warning-signs.md` — 漏桶场景及修复方案

**大小：** 619 行

### 技能 #2：`saas-economics-efficiency-metrics`（组件）

**目的：** 判断您的 SaaS 商业模式是否根本可行且资本高效。

**覆盖范围（17 项指标）：**

**单位经济学（8 项）：**
- 毛利率
- CAC（客户获取成本）
- LTV（生命周期价值）
- LTV:CAC 比率
- 回收期
- 贡献利润率
- 毛利率回收期
- 按渠道的 CAC 回收期

**资本效率（5 项）：**
- 烧钱率
- 跑道
- 运营费用（OpEx）
- 净收入（利润率）
- 营运资金影响

**效率比率（4 项）：**
- 40 法则
- 魔法数字
- 经营杠杆
- 单位经济学

**关键概念：**
- 单位经济学家族（CAC、LTV、回收期、利润率）
- 资本效率家族（烧钱、跑道、OpEx）
- 效率比率（40 法则、魔法数字）
- 反模式（忽视回收期、在未检查现金流的情况下庆祝高 LTV）

**支持文件：**
- `template.md` — 综合单位经济学计算器
- `examples/healthy-unit-economics.md` — 盈利性扩张示例
- `examples/cash-trap.md` — LTV:CAC 良好但回收期极差的警告

**大小：** 684 行

### 技能 #4：`feature-investment-advisor`（交互式）

**目的：** 指导产品经理根据财务影响分析评估是否构建某项功能。

**交互流程（4 步）：**
1. **收入关联** — 直接变现 vs. 间接（留存/转化/扩展）
2. **成本结构** — 开发成本 + 销货成本 + 运营费用影响
3. **约束条件** — 竞争威胁、能力限制、依赖关系
4. **建议** — 立即构建 / 出于战略原因构建 / 不构建 / 稍后构建

**建议模式：**
- ✅ **立即构建：** 强劲的投资回报率（直接 >3:1，留存 >10:1），正向利润率
- ⚠️ **出于战略原因构建：** 投资回报率弱但战略价值高
- 🚫 **不构建：** 负投资回报率，稀释利润率，无战略价值
- 🔄 **稍后构建：** 高不确定性，需要先验证

**主要特点：**
- 基于收入类型的自适应分支
- 包含敏感性分析的完整投资回报率计算
- 利润率影响分析（贡献利润率）
- 回收期计算
- 功能间比较

**支持文件：**
- `examples/conversation-flow.md` — 完整的时间跟踪附加组件评估

**大小：** 615 行

### 技能 #5：`acquisition-channel-advisor`（交互式）

**目的：** 指导产品经理评估是否扩展、测试或终止某个获客渠道。

**交互流程（4 步）：**
1. **单位经济学** — 按渠道的 CAC、LTV、LTV:CAC 比率、回收期
2. **客户质量** — 按渠道的队列留存、NRR、ICP 匹配度
3. **可扩展性** — 魔法数字、可触达量、CAC 趋势
4. **建议** — 积极扩展 / 测试与优化 / 终止或暂停 / 战略性投资

**建议模式：**
- ✅ **积极扩展：** LTV:CAC >3:1，回收期 <12 个月，高质量，魔法数字 >0.75
- 🧪 **测试与优化：** 边际经济学（2-3:1），可修复的问题
- 🚫 **终止或暂停：** LTV:CAC <1.5:1，无改善路径
- 🎯 **战略性投资：** 经济学差但战略价值高（限制支出）

**主要特点：**
- 多渠道比较表
- 预算重新分配建议
- CAC 趋势分析（改善、稳定、恶化）
- 客户质量与数量的权衡
- 魔法数字效率评分

**支持文件：**
- `examples/conversation-flow.md` — 内容营销渠道评估（LTV:CAC 16:1）

**大小：** 619 行

### 技能 #6：`finance-based-pricing-advisor`（交互式）

**目的：** 使用 ARPU/ARPA 分析、转化率、流失风险和 NRR 影响来评估定价变更的财务影响。

**范围：**
- **这是什么：** 定价决策的财务影响评估
- **这不是什么：** 基于价值的定价、WTP 研究、包装架构（参见未来第 8 阶段）

**交互流程（4 步）：**
1. **定价变更类型** — 提价、新层级、附加组件、基于使用量、折扣、包装
2. **预期影响** — ARPU 提升、转化率变化、流失风险、扩展潜力
3. **当前状态** — 基线指标（MRR、ARPU、流失率、NRR、CAC）
4. **建议** — 广泛实施 / 先测试（A/B） / 修改方案 / 不要变更

**建议模式：**
- ✅ **广泛实施：** 净收入明确为正，流失风险低
- 🧪 **先测试（A/B）：** 影响不确定，需要在广泛推出前验证
- ⚠️ **修改方案：** 原始提案风险高，存在更好的替代方案
- 🚫 **不要变更：** 风险大于收益，关注其他方面

**主要特点：**
- 按定价变更类型的自适应分支
- 流失建模（考虑流失损失后的净收入）
- CAC 回收期影响计算
- 敏感性分析（乐观/悲观/盈亏平衡场景）
- 老客户保护策略建议

**大小：** 739 行

### 技能 #7：`business-health-diagnostic`（交互式）

**目的：** 通过综合分析增长、留存、单位经济学和资本效率来诊断整体 SaaS 业务健康状况。

**交互流程（4 步）：**
1. **增长与留存** — 收入增长、NRR、流失率、速动比率、队列趋势
2. **单位经济学** — CAC、LTV、LTV:CAC、回收期、毛利率
3. **资本效率** — 烧钱率、跑道、40 法则、魔法数字、经营杠杆
4. **综合诊断** — 带有优先级建议的健康记分卡

**诊断模式：**
- ✅ **健康：** 所有维度强劲，已识别优化机会
- ⚠️ **中等：** 1-2 个可修复问题，明确的 90 天行动计划
- 🚨 **令人担忧：** 多个紧急问题，90 天生存计划
- 🚨🚨 **危急：** 存在性危机，需要紧急行动

**主要特点：**
- 阶段特定基准（早期 / 增长 / 规模）
- 危险信号分类（关键 / 高 / 中优先级）
- 优先行动（前 1-3 个问题及时间表）
- 领先指标（不仅仅是滞后指标）
- 协调来自技能 #1-3 的所有 32 项指标

**大小：** 758 行

## 决策框架

### 框架 1：我们应该构建这个功能吗？
**技能：** `feature-investment-advisor`

**关键问题：**
1. 收入关联？（直接变现还是间接留存/转化/扩展）
2. 成本结构？（开发成本、销货成本、运营费用）
3. 投资回报率计算？（收入影响 / 投资）
4. 战略价值？（竞争护城河、平台赋能、市场定位）

**决策标准：**
- **构建条件：** 投资回报率 >3:1（直接）或 LTV 影响 >10:1（留存）或战略价值压倒一切
- **不构建条件：** 贡献利润率为负，回收期超过客户生命周期

**使用的指标：** 收入、ARPU、毛利率、贡献利润率、LTV

---

### 框架 2：我们应该扩展这个渠道吗？
**技能：** `acquisition-channel-advisor`

**关键问题：**
1. 单位经济学？（CAC、LTV、LTV:CAC 比率、回收期）
2. 客户质量？（按渠道的队列留存、NRR、ICP 匹配度）
3. 可扩展性？（魔法数字、可触达量、CAC 趋势）
4. 战略契合度？（细分市场匹配、销售模式兼容性）

**决策标准：**
- **扩展条件：** LTV:CAC >3:1 且回收期 <18 个月且质量良好且魔法数字 >0.75
- **终止条件：** LTV:CAC <1.5:1 且无明确改善路径

**使用的指标：** CAC、LTV、LTV:CAC、回收期、NRR、魔法数字、流失率

---

### 框架 3：我们应该改变定价吗？
**技能：** `finance-based-pricing-advisor`

**关键问题：**
1. ARPU/ARPA 影响？（每客户收入会增加吗？）
2. 转化率影响？（帮助还是损害试用到付费的转化？）
3. 流失风险？（现有客户会离开吗？）
4. NRR 影响？（促进扩展还是导致收缩？）
5. CAC 回收期影响？（定价变更是否影响单位经济学？）

**决策标准：**
- **实施条件：** 考虑流失风险后净收入影响为正，可以先对细分市场进行测试
- **不变更条件：** 流失风险高且无抵消性扩展，无法验证假设

**使用的指标：** ARPU、ARPA、流失率、NRR、CAC 回收期、转化率

---

### 框架 4：业务健康吗？
**技能：** `business-health-diagnostic`

**关键问题：**
1. 增长与留存？（收入增长、NRR、流失率、速动比率）
2. 单位经济学？（LTV:CAC、回收期、毛利率）
3. 资本效率？（40 法则、魔法数字、跑道）
4. 阶段适当基准？（早期 vs. 增长 vs. 规模）

**决策标准：**
- **健康：** 所有维度达到阶段基准，趋势改善
- **令人担忧：** 多个危险信号，需要紧急行动

**使用的指标：** 综合 4 个维度的所有 32 项指标

## 阶段特定基准

### 早期阶段（ARR 低于 1000 万美元）
**重点：** 产品市场契合度，单位经济学

| 指标 | 基准 |
|--------|-----------|
| 增长率 | >50% 同比 |
| LTV:CAC | >3:1 |
| 毛利率 | >70% |
| 跑道 | >12 个月 |
| 可接受 | 负利润率、高烧钱率（如果单位经济学可行） |

### 增长阶段（ARR 1000万-5000万美元）
**重点：** 高效扩展

| 指标 | 基准 |
|--------|-----------|
| 增长率 | >40% 同比 |
| NRR | >100% |
| 40 法则 | >40 |
| 魔法数字 | >0.75 |
| 可接受 | 如果增长强劲，中等烧钱率 |

### 规模阶段（ARR 5000万美元以上）
**重点：** 盈利能力，效率

| 指标 | 基准 |
|--------|-----------|
| 增长率 | >25% 同比 |
| NRR | >110% |
| 40 法则 | >40 |
| 利润率 | >10% |
| 必需 | 正向或接近正向的现金流 |

## 如何使用该套件

### 用于季度业务评审

**工作流程：**
1. 从 `business-health-diagnostic` 开始 — 获取综合健康记分卡
2. 从诊断中识别前 1-3 个优先事项
3. 使用决策顾问解决优先事项：
   - 留存问题？ → 使用 `feature-investment-advisor` 评估留存功能
   - 获客问题？ → 使用 `acquisition-channel-advisor` 优化渠道
   - 变现问题？ → 使用 `finance-based-pricing-advisor` 进行定价变更
4. 每周/每月使用 `saas-revenue-growth-metrics` 和 `saas-economics-efficiency-metrics` 进行监控

### 用于功能优先级排序

**工作流程：**
1. 审查 `saas-revenue-growth-metrics` — 了解当前 ARPU、流失率、NRR
2. 审查 `saas-economics-efficiency-metrics` — 了解 LTV、贡献利润率
3. 使用 `feature-investment-advisor` — 评估功能投资回报率
4. 使用投资回报率计算并排比较功能

### 用于渠道预算分配

**工作流程：**
1. 使用 `saas-economics-efficiency-metrics` 计算各渠道的 CAC、LTV、回收期
2. 对每个渠道使用 `acquisition-channel-advisor`
3. 获取带有建议的多渠道比较表
4. 将预算从低投资回报率渠道重新分配到高投资回报率渠道

### 用于董事会/投资者会议

**工作流程：**
1. 运行 `business-health-diagnostic` — 获取综合记分卡
2. 使用 `finance-metrics-quickref` — 准备关键指标幻灯片
3. 参考决策顾问，用于“本季度你们做了什么？”的叙述：
   - 已构建的功能 → `feature-investment-advisor` 的投资回报率
   - 已扩展的渠道 → `acquisition-channel-advisor` 的效率
   - 定价变更 → `finance-based-pricing-advisor` 的影响

## 常见用例

### 用例 1：“我们应该构建暗黑模式吗？”

**技能：** `feature-investment-advisor`

**步骤：**
1. 收入关联：间接（假设能改善留存）
2. 成本结构：8 万美元开发成本
3. 证据：50 个功能请求，无流失数据
4. 建议：稍后构建 — 先验证对流失的影响

### 用例 2：“我们应该将谷歌广告支出扩大 2 倍吗？”

**技能：** `acquisition-channel-advisor`

**步骤：**
1. 单位经济学：CAC 800 美元，LTV 2000 美元（2.5:1 比率）
2. 客户质量：前 90 天流失率较高
3. 可扩展性：魔法数字 0.6（边际）
4. 建议：在扩展前进行测试与优化

### 用例 3：“我们应该提价 20% 吗？”

**技能：** `finance-based-pricing-advisor`

**步骤：**
1. 定价变更：仅对新客户提价 20%
2. 预期影响：ARPU 提升 20%，转化率下降 <5%
3. 流失风险：低（现有客户受保护）
4. 建议：实施 — 风险低，净收益为正

### 用例 4：“我们的业务健康吗？”

**技能：** `business-health-diagnostic`

**步骤：**
1. 增长：同比增长 40%，NRR 95%
2. 经济学：LTV:CAC 3.5:1，回收期 12 个月
3. 效率：40 法则 = 38，跑道 12 个月
4. 诊断：中等 — 在扩展前修复留存问题（NRR <100%）

### 定价与变现策略套件（计划 7 项技能）

**当前套件未涵盖的全面定价主题：**

- 定价是一个庞大的领域（心理学、策略、研究、测试）
- 当前套件专注于财务影响分析（评估提议的变更）
- 未来套件将涵盖全面的定价策略（从头开始设计）

# Marketplace Submission Runbook（面向 PM）

这个 runbook 是给产品经理准备的，帮助你用一种实用、低压力的方式把 skills 发布到 marketplace。

## 何时使用

当满足以下情况时，就用这份 runbook：
- 你的 skill library 已经准备好对外分享
- 你希望分发步骤可以重复执行
- 你需要一套不依赖深度技术设置、队友也能执行的工作流

## Step 1：定义你的发布目标

这一轮只选一个主要目标：

- Discoverability（让更多人能发现你的 library）
- Credibility（进入可信的精选渠道）
- Adoption（提升安装量 / 使用量）
- Learning（收集结构化反馈）

写下一句话：

`This launch cycle is successful if _______.`

## Step 2：一次性准备好提交包

准备这些可复用素材：

- 一段式描述
- 一个简短 tagline
- 3-5 个 featured skills，每个带一句价值说明
- 安装说明
- 许可证声明
- 联系方式 / 反馈渠道

## Step 3：优先选择 3 个 Marketplace 目标

按这个顺序：

1. 一个低摩擦目录站点（快速拿到成果）
2. 一个精选社区清单（提升可信度）
3. 一个官方 / 门槛更高的渠道（提升正当性）

## Step 4：执行提交

对每个目标：

1. 提交 listing 或 PR
2. 把 URL 记到 tracking log
3. 记录状态：`submitted`、`in review`、`live`、`needs changes`
4. 设定 follow-up 日期（通常是 5-7 天后）

## Step 5：用实用角度做一次公告

发一条公告，至少包含：

- 一个 PM 痛点
- 一个具体 use case
- 一个 featured skill 示例
- 一个直接的 “how to use/install” 步骤

## Step 6：衡量并迭代

每周记录：

- Listing views（如果拿得到）
- Repo stars / watchers
- Installs 或 usage proxies
- Questions / feedback themes

然后在下一周前只做一项改进：

- 文案更紧
- Featured skill 顺序更合理
- 示例更好
- Onboarding 说明更清楚

## 每一轮的完成定义

- [ ] 已提交 3 个优先 marketplace
- [ ] 已发布 1 条公告
- [ ] tracking table 已更新
- [ ] 已根据反馈上线至少 1 项改进

## 给非技术团队的说明

整个流程都可以只在浏览器 UI 里完成。

CLI / automation 是可选项，不是必须项。

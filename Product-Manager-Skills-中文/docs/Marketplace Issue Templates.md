# Marketplace Issue 模板

将这些模板放入你自己的仓库，用于跟踪 marketplace 分发工作。

## 模板 1：提交到精选社区清单

**标题**
`Submit <repo-name> to <curated-list-name>`

**正文**
```md
## 目标
将我们的技能库添加到 <curated-list-name>。

## 重要性
- 提升可信度和可发现性
- 触达正在浏览精选技能的用户

## 建议的列表条目
<插入一行列表文本>

## 任务
- [ ] Fork 目标仓库（如果是基于 PR 的）
- [ ] 创建分支
- [ ] 在正确的部分添加列表条目
- [ ] 提交 PR 并附上简要理由
- [ ] 回应维护者反馈
- [ ] 合并并记录最终链接

## 完成标准
- PR/列表已上线
- 链接已记录在 marketplace 跟踪日志中
```

## 模板 2：加入 / 申请 Marketplace

**标题**
`Apply to <marketplace-name> (manual submission)`

**正文**
```md
## 目标
将我们的库提交到 <marketplace-name>。

## 所需输入
- 名称
- 联系邮箱
- 库 URL
- 简短描述

## 任务
- [ ] 打开提交表单
- [ ] 填写所需详情
- [ ] 提交表单
- [ ] 获取确认信息（文本/截图）
- [ ] 在跟踪日志中更新状态和后续跟进日期

## 完成标准
- 提交已确认
- 证据已记录在此 issue 中
```

## 模板 3：改进 Marketplace 可发现性元数据

**标题**
`Update repository metadata for marketplace discoverability`

**正文**
```md
## 目标
提高在 marketplace 和 GitHub 搜索中的可发现性。

## 范围
- 仓库主题/标签
- 描述文本
- 精选技能排序

## 任务
- [ ] 更新仓库主题
- [ ] 优化简短描述
- [ ] 验证更改已公开可见
- [ ] 在此 issue 中记录更改前后的快照

## 完成标准
- 元数据更新已上线并已验证
```

## 下一步行动（负责人 + 日期）

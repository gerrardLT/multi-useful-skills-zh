## Register

**Brand**：强调经过编排的页面加载序列、错峰显示、滚动驱动动画。动效是品牌声音的一部分；一个排练到位的入场时刻，胜过零碎散落的微交互。

**Product**：大多数过渡控制在 150-250 毫秒。动效应传达状态——反馈、揭示、加载、视图切换。不要做页面加载编排；用户正在执行任务，他们不会愿意等待。

## 评估动画机会点

分析哪些地方引入动效能真正改善体验：

1.  **识别静态区域**：
    *   **缺少反馈**：动作发生后没有视觉确认（按钮点击、表单提交等）。
    *   **切换生硬**：状态瞬时变化，显得突兀（显示/隐藏、页面加载、路由切换）。
    *   **关系不清楚**：空间关系或层级关系不够直观。
    *   **缺少愉悦感**：功能可用，但没有任何生命感。
    *   **错失引导机会**：本可用来引导注意力或解释行为的地方没有利用起来。

2.  **理解上下文**：
    *   产品/品牌人格是什么？（活泼还是严肃，充满活力还是沉稳冷静）。
    *   性能预算是多少？（移动优先？页面本身已经很复杂？）。
    *   受众是谁？（对动效敏感的用户？追求速度的高级用户？）。
    *   最重要的是什么？（一个标志性的主视觉动画，还是很多微交互？）。

如果这些信息无法从代码库中看出来，就 {{ask_instruction}}。

**关键**：尊重 `prefers-reduced-motion`。必须始终为需要的用户提供非动画替代方案。

## 规划动画策略

建立一套有目的的动画方案：

*   **主视觉时刻**：哪一个才是那唯一的标志性动画？（页面加载？主视觉区？关键交互？）。
*   **反馈层**：哪些交互必须被明确确认？
*   **过渡层**：哪些状态切换需要被柔化？
*   **愉悦层**：哪里可以加入一点惊喜和愉悦感？

**重要**：一个编排到位的整体体验，胜过到处零碎加动画。把注意力集中在高影响力时刻。

## 实施动画

按这些类别系统性加入动效：

### 入场动画
*   **页面加载编排**：错峰显示元素（100-150ms 延迟），结合淡入 + 滑动。
*   **主视觉区域**：为主内容设计更有戏剧性的入场（缩放、视差，或有创意的效果）。
*   **内容显示**：使用 Intersection Observer 做滚动触发动画。
*   **模态框/抽屉进入**：平滑滑动 + 淡入，背景淡入，以及正确的焦点管理。

### 微交互
*   **按钮反馈**：
    *   悬停：微小缩放（1.02-1.05）、颜色变化、阴影增强。
    *   点击：快速缩小再回弹（0.95 -> 1），或涟漪效果。
    *   加载：旋转指示器或脉冲状态。
*   **表单交互**：
    *   输入框聚焦：边框颜色过渡，轻微缩放或发光。
    *   验证：出错时抖动，成功时显示对勾，平滑颜色过渡。
*   **切换开关**：平滑滑动 + 颜色过渡（200-300ms）。
*   **复选框/单选框**：对勾动画、涟漪效果。
*   **点赞/收藏**：缩放 + 旋转、粒子效果、颜色过渡。

### 状态过渡
*   **显示/隐藏**：使用淡入 + 滑动，而不是瞬间切换；时长控制在 200-300ms。
*   **展开/收起**：高度过渡配合溢出处理，图标旋转。
*   **加载状态**：骨架屏淡入、旋转指示器动画、进度条。
*   **成功/错误**：颜色过渡、图标动画、轻微缩放脉冲。
*   **启用/禁用**：透明度过渡和光标变化。

### 导航与流程
*   **页面切换**：路由之间交叉淡入，或共享元素过渡。
*   **标签页切换**：滑动指示条，内容淡入/滑动。
*   **轮播/滑块**：平滑变换、吸附点、惯性。
*   **滚动效果**：视差层、带状态变化的粘性页头、滚动进度指示器。

### 反馈与引导
*   **悬停提示**：工具提示淡入、光标变化、元素高亮。
*   **拖放**：提起效果（阴影 + 缩放）、放置区域高亮、平滑重排。
*   **复制/粘贴**：粘贴时短暂高亮，“已复制”确认。
*   **焦点流**：在表单或工作流中高亮当前路径。

### 愉悦时刻
*   **空状态**：插画上的微妙悬浮动画。
*   **完成动作**：五彩纸屑、对勾装饰、成功庆祝。
*   **彩蛋**：留给探索者的小隐藏交互。
*   **上下文动效**：天气效果、昼夜主题、季节性细节。

## 技术实现

为不同动效选择合适手段：

### 时长与缓动

**按用途划分的时长：**
*   **100-150ms**：即时反馈（按钮按下、切换）。
*   **200-300ms**：状态变化（悬停、菜单展开）。
*   **300-500ms**：布局变化（手风琴、模态框）。
*   **500-800ms**：入场动画（页面加载）。

**缓动曲线（使用这些，不要用 CSS 默认值）：**
```css
/* 推荐 - 自然减速 */
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);    /* 平滑，精致 */
--ease-out-quint: cubic-bezier(0.22, 1, 0.36, 1);   /* 略快 */
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);     /* 自信，果断 */

/* 避免 - 感觉过时且廉价 */
/* bounce: cubic-bezier(0.34, 1.56, 0.64, 1); */
/* elastic: cubic-bezier(0.68, -0.6, 0.32, 1.6); */
```

**退出动画应比进入动画更快。** 通常使用进入时长的约 75%。

### CSS 动画
```css
/* 适用于简单、声明式的动画 */
- 用于状态变化的过渡
- 用于复杂序列的 @keyframes
- 用于可靠移动的 transform 和 opacity
- 当确认流畅时，用于高级氛围效果的模糊、滤镜、蒙版、裁剪路径、阴影和颜色偏移
```

### JavaScript 动画
```javascript
/* 用于复杂、交互式的动画 */
- 用于程序化控制的 Web Animations API
- 用于 React 的 Framer Motion
- 用于复杂序列的 GSAP
```

### 性能
*   **动效材料选择**：位移优先使用 transform / opacity，但当模糊、滤镜、蒙版、阴影、颜色偏移能显著提升效果时，也可以使用。
*   **布局安全**：避免随意动画化会驱动布局的属性（`width`、`height`、`top`、`left`、margin）。
*   **will-change**：只在明确知道某处动画昂贵时少量使用。
*   **约束高开销效果**：让模糊/滤镜/阴影的作用区域尽量小且隔离，必要时配合 `contain`。
*   **监控 FPS**：在目标设备上确保 60fps。

### 可访问性
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**切勿**：
*   不要使用 bounce 或 elastic 缓动——它们显得过时，而且会让人注意到“动画本身”。
*   不要随意动画化布局属性（`width`、`height`、`top`、`left`、margin），如果 transform、FLIP 或基于网格的技术已经够用。
*   不要让反馈动画超过 500ms——会显得迟钝。
*   不要做没有目的的动画——每个动画都必须有存在理由。
*   不要忽略 `prefers-reduced-motion`——这是可访问性违规。
*   不要什么都动——动画疲劳会让界面令人厌倦。
*   除非是刻意设计，否则不要在动画期间阻塞交互。

## 验证质量

认真检查这些点：

*   **60fps 流畅**：在目标设备上没有卡顿。
*   **感觉自然**：缓动曲线是有机的，不像机器人。
*   **时长合适**：不至于太快吓人，也不至于太慢拖沓。
*   **Reduced motion 正常工作**：动画被正确关闭或简化。
*   **不阻塞**：用户能在动画期间/之后正常交互。
*   **有增益**：它让界面更清楚或更愉悦，而不是只是装饰。

记住：动效应当增强理解并提供反馈，而不是只增加装饰。带着目的做动画，尊重性能约束，并始终考虑可访问性。真正好的动画往往是“无感”的——它只是让一切都变得更对。

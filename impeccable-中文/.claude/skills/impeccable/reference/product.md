# Product register

当设计是为产品服务时，就进入这个 register：应用界面、后台仪表盘、设置面板、数据表格、工具类界面、登录后的使用场景，凡是用户在“完成任务”时所面对的界面，都属于这里。

## 产品界面的 slop 测试

测试标准不是“别人会不会一眼说这是 AI 做的”，因为在这里，熟悉感本来就常常是优点。真正的问题是：一个深度使用过该品类优秀工具的人（比如 Linear、Figma、Notion、Raycast、Stripe），坐下来面对这个界面时，会立刻产生信任，还是会在每一个微妙不对劲的组件前停顿一下？

产品 UI 的失败方式不是“太平”，而是“莫名其妙但没有意义”：装饰过度的按钮、彼此不搭的表单控件、无必要的动效、本该用于标签的位置却用了展示型字体、为了标准任务硬造的新交互。这里的标准是“经过打磨后的熟悉感”。工具应该融进任务本身，而不是跳出来刷存在感。

## 排版

- **系统字体完全合理。** `-apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif` 能在各个平台上带来原生感。Inter 之所以常见，是因为它确实是一个稳妥的跨平台默认项。
- **很多时候只用一个字体家族就够了。** 产品界面通常不需要 display/body 搭配。一个调得好的 sans 就能同时承担标题、按钮、标签、正文和数据。
- **使用固定 rem 比例，而不是 fluid scale。** `clamp()` 式标题不适合产品 UI。用户通常以稳定 DPI 使用它，而一个在侧栏里会继续缩小的 fluid h1 往往更糟。
- **字号比例更紧一些。** 相邻层级在 1.125 到 1.2 之间通常就够了。产品界面里的文字元素比品牌页面多得多，过于夸张的字体对比只会制造噪音。
- **行长规则对长文依然成立**（65 到 75ch）。但数据和紧凑 UI 可以更密，例如表格做到 120ch 以上完全没问题。

## 颜色

产品默认应落在 Restrained。某一个特定界面可以争取 Committed，比如一个通过类别色承载报告结构的仪表盘，或一个开屏就强烈浸润色彩的新手引导流程，但 Restrained 仍然是底线。

- 要有一套状态丰富的语义色词汇：hover、focus、active、disabled、selected、loading、error、warning、success、info。并把它们标准化。
- 强调色只用于主操作、当前选择和状态指示，不要把它拿来做装饰。
- 需要第二层中性色，用于 sidebar、toolbar 和 panel，色温可以比内容区表面略冷或略暖。

## 布局

- 网格要可预测。一致性本身就是 affordance，用户越能预期结构，导航速度就越快。
- 熟悉的模式本身就是功能。标准导航（顶栏、侧栏）、面包屑、标签页、表单布局都有成熟预期，不要为了“有味道”而重造。
- 响应式变化应该是结构性的，例如折叠侧边栏、响应式表格、按断点调整列数，而不是靠 fluid typography 来硬撑。

## 组件

每个交互组件都必须具备：default、hover、focus、active、disabled、loading、error。不要只做了一半就上线。

- 加载状态优先用 skeleton，而不是在内容中央丢一个 spinner。
- 空状态应该教会用户怎么使用界面，而不是只写一句“这里什么都没有”。
- 整个产品表面必须保持一致 affordance。按钮形状一致、表单控件语言一致、图标风格一致。

## 动效

- 大多数过渡保持在 150 到 250ms。用户正处于工作流里，不要让他们等你演完。
- 动效用来传达状态，不是用来装饰。状态变化、反馈、加载、揭示，这些可以；其他的一般不需要。
- 不要做整页上来的编排式入场动画。产品界面是让人立刻进入任务的，不是让人看它怎么出现的。

## 产品界面禁令（在共享的绝对禁令之外）

- 不传达状态的装饰性动效
- 跨页面组件词汇不一致。如果“保存”按钮在两个地方长得不一样，其中一个就是错的
- 在 UI 标签、按钮、数据区域使用展示型字体
- 为了追求风格而重造标准 affordance（自定义滚动条、怪异表单控件、非标准 modal）
- 在非激活状态上使用过重色彩或高饱和强调色

## 产品界面允许的做法

产品界面可以做一些品牌页面不适合做的事。

- 使用系统字体，以及熟悉的 sans 默认项（Inter、SF Pro、system-ui 栈）
- 使用标准导航模式：顶栏 + 侧边栏、面包屑、标签页、命令面板
- 接受信息密度：大量行的表格、很多标签的面板、在用户确实需要时展示高密度信息
- 优先一致性而不是惊喜。跨页面保持同一视觉词汇是美德；真正的惊喜感应该留给关键时刻，而不是每一页

# impeccable-中文 目录结构说明

## 项目定位

`impeccable-中文` 是一个偏前端审美与界面质量提升的技能仓库，同时也包含官网、浏览器扩展、可分发插件、服务端下载接口与大量测试夹具。

## 顶层目录与文件

- `README.md`
  - 仓库总说明。
- `README.npm.md`
  - npm 分发视角说明。
- `PRODUCT.md`
  - 产品定位。
- `DESIGN.md`
  - 设计原则。
- `DEVELOP.md`
  - 开发说明。
- `HARNESSES.md`
  - 运行/兼容环境说明。
- `NOTICE.md`
  - 引用与许可补充。
- `DESIGN.json`
  - 结构化设计数据。
- `package.json`
  - 依赖与脚本。
- `wrangler.toml`
  - Worker/部署配置。
- `content/`
  - 站点内容源。
- `public/`
  - 最终静态站点资源。
- `source/`
  - 源 skill 内容。
- `plugin/`
  - 可分发 skill 版本。
- `extension/`
  - 浏览器扩展。
- `server/`
  - 服务端逻辑。
- `functions/`

- `tests/`

## 重点子目录内文件

### `source/skills/impeccable/`

这是最原始的 skill 源目录。

- `SKILL.md`
  - 技能主文档。
- `reference/*.md`
  - 分主题参考文档，如：
  - `typography.md`
  - `motion-design.md`
  - `responsive-design.md`
  - `color-and-contrast.md`
  - `brand.md`
  - `craft.md`
- `scripts/live.mjs`
  - live 模式入口。
- `scripts/live-server.mjs`
  - 本地 live server。
- `scripts/live-inject.mjs`
  - 页面注入逻辑。
- `scripts/live-accept.mjs`
  - 接收确认流程。
- `scripts/design-parser.mjs`
  - 设计解析。
- `scripts/detect-csp.mjs`
  - CSP 探测。
- `scripts/cleanup-deprecated.mjs`
  - 清理旧内容。

### `plugin/skills/impeccable/`

这是分发后的 skill 目录，结构和 `source/skills/impeccable/` 基本对应：

- `SKILL.md`
- `reference/*.md`
- `scripts/*.mjs`

如果你改的是技能本体，通常先看 `source/`，再确认 `plugin/` 是否需要同步。

### `content/site/`

这是站点内容源：

- `skills/*.md`
  - 各个技能的站点内容页。
- `tutorials/getting-started.md`
  - 新手教程。
- `tutorials/iterate-live.md`
  - live 迭代教程。
- `tutorials/critique-with-overlay.md`
  - 叠层批注类教程。
- `partials/header.html`
  - 公共模板片段。
- `anti-patterns-catalog.js`
  - 反模式目录数据。

### `public/`

这是最终静态资源目录。

核心页面与资源：

- `index.html`
  - 首页。
- `app.js`
  - 前端总入口脚本。
- `css/main.css`
  - 主样式。
- `css/tokens.css`
  - 设计变量。
- `css/live-mode.css`
  - live 模式样式。
- `js/components/*`
  - 站点组件脚本。
- `js/effects/*`
  - 动效脚本。
- `js/demos/skills/*`
  - 技能演示脚本。

专题页面与案例：

- `designing/index.html`
- `live-mode/index.html`
- `cases/neo-mirai/index.html`
- `neo-mirai/index.html`

辅助资源：

- `assets/*`
- `antipattern-images/*`
- `antipattern-examples/*`

### `tests/`

这个目录很大，但可按用途理解：

- `detect-antipatterns*.test.*`
  - 反模式检测测试。
- `live-*.test.*`
  - live 模式测试。
- `server/*.test.js`
  - 服务端测试。
- `framework-fixtures/`
  - 不同前端框架夹具。
- `fixtures/antipatterns/`
  - 正反例样本。

## Skills 简要说明

- `impeccable`
  - 面向前端界面设计、重设计、评审、打磨和实时迭代的综合技能；覆盖视觉层级、排版、颜色、布局、动效、UX 文案、响应式、反模式检查和设计系统抽取，不适用于纯后端任务。

## 目录关系

- `source/` 是技能源头。
- `plugin/` 是分发产物层。
- `content/` 是站点内容源。
- `public/` 是站点最终静态输出。
- `scripts/` 负责从内容源生成站点、资源和扩展。
- `extension/` 是浏览器交付层。
- `server/` 与 `functions/` 是服务端支撑层。
- `src/` 与 `tests/` 负责检测能力与验证能力。

## 建议阅读顺序

1. `README.md`
2. `PRODUCT.md`
3. `DESIGN.md`
4. `DEVELOP.md`
5. `source/skills/impeccable/SKILL.md`
6. `content/site/`
7. `public/`、`extension/`、`scripts/`

## 常见工作路径

- 想改 skill 本体：
  - 先看 `source/skills/impeccable/`
- 想改官网内容：
  - 看 `content/site/`
- 想改最终页面效果：
  - 看 `public/`
- 想改扩展：
  - 看 `extension/`
- 想改下载接口：
  - 看 `server/` 和 `functions/`
- 想改构建：
  - 看 `scripts/`
- 想改检测规则：
  - 看 `src/` 和 `tests/fixtures/antipatterns/`

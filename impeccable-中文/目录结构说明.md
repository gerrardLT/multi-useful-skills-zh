# impeccable-中文 目录结构说明

## 项目定位

`impeccable-中文` 是一个面向前端设计与审美提升的技能仓库，同时包含网站内容、浏览器扩展、可发布插件、源 skill 版本、构建脚本和检测能力。

## 顶层目录说明

- `.agents/`、`.claude/`、`.gemini/`、`.opencode/`、`.cursor/`、`.kiro/` 等
  - 面向不同 AI 工具或平台的接入配置。
- `.claude-plugin/`
  - Claude 插件相关目录。
- `bin/`
  - 命令行入口文件。
- `content/`
  - 站点内容源。
  - 包括教程、技能文案、局部模板等。
- `docs/`
  - 设计决策、补充说明文档。
- `extension/`
  - 浏览器扩展代码与上架相关资料。
- `functions/`
  - 云函数或 API 相关入口。
- `lib/`
  - 共享逻辑与底层工具。
- `plugin/`
  - 可分发插件形态的 skill 内容。
- `public/`
  - 站点静态资源、页面、案例页、演示页。
- `scripts/`
  - 构建、打包、生成、转换脚本。
- `server/`
  - 本地或服务端逻辑。
- `source/`
  - 最原始的 skill 内容源。
  - 一般是发布到 `plugin/`、站点或其他平台前的主编辑区。
- `src/`
  - 项目源码实现。
- `tests/`
  - 测试与测试夹具。
- `tools/`
  - 辅助开发工具。

## 关键文件说明

- `README.md`
  - 项目主说明与使用入口。
- `README.npm.md`
  - npm 包分发视角的说明。
- `AGENTS.md`
  - 代理协作说明。
- `CLAUDE.md`
  - Claude 环境下的维护和行为规则。
- `PRODUCT.md`
  - 产品定位、品牌个性、设计目标。
- `DESIGN.md`
  - 设计系统与风格约束。
- `DEVELOP.md`
  - 开发、构建与扩展方式说明。
- `HARNESSES.md`
  - 各类工具/平台兼容性与 harness 说明。
- `NOTICE.md`
  - 许可证来源与引用说明。
- `DESIGN.json`
  - 结构化设计数据。
- `package.json`
  - 包信息、脚本入口、依赖配置。

## 目录关系

- `source/` 是内容源头。
  - 这里的 skill 与 reference 通常是维护时的主要编辑对象。
- `plugin/` 是分发形态之一。
  - 往往由 `source/` 构建或同步而来。
- `content/` + `public/`
  - 共同构成站点内容层。
  - `content/` 更偏源内容，`public/` 更偏最终页面和静态资源。
- `src/`、`lib/`、`server/`、`functions/`
  - 共同构成功能实现层。
- `extension/`
  - 独立的浏览器扩展交付层。
- `scripts/`、`tools/`
  - 负责生成、打包、辅助开发。

## 建议阅读顺序

1. `README.md`
2. `PRODUCT.md`
3. `DESIGN.md`
4. `DEVELOP.md`
5. `source/skills/impeccable/`
6. 需要站点内容时看 `content/` 与 `public/`
7. 需要实现细节时看 `src/`、`lib/`、`scripts/`、`tests/`

## 常见工作路径

- 想改设计原则：
  - 优先看 `PRODUCT.md`、`DESIGN.md`、`source/skills/impeccable/`
- 想改技能内容：
  - 优先改 `source/`
  - 再确认 `plugin/` 和站点是否需要同步
- 想改站点页面或案例：
  - 看 `content/`、`public/`
- 想改扩展：
  - 看 `extension/`
- 想改构建流程：
  - 看 `scripts/`、`DEVELOP.md`、`package.json`

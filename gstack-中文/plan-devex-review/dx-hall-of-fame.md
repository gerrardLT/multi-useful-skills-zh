# DX 名人堂参考

仅阅读当前审核通过的部分。不要加载整个文件。

## 第 1 关：入门

**黄金标准：**
- **Stripe**：7 行代码用于为卡充电。登录时，文档会预先填写您的测试 API 密钥。Stripe Shell 在文档页面内运行 CLI。无需本地安装。
- **Vercel**：`git push` = 使用 HTTPS 的全球 CDN 上的实时站点。每个 PR 都会获得预览 URL。一个 CLI 命令：`vercel`。
- **职员**：`<SignIn />`、`<SignUp />`、`<UserButton />`。 3 个 JSX 组件，可通过电子邮件、社交、MFA 进行开箱即用的工作身份验证。
- **Supabase**：创建 Postgres 表，立即自动生成 REST API + 实时 + 自记录文档。
- **Firebase**：`onSnapshot()`。 3 条线路可在所有客户端之间实时同步，并内置离线持久性。
- **Twilio**：控制台中的虚拟电话。发送/receive短信，无需购买号码，无需信用卡。结果：激活度提高了 62%。

**反模式：**
- 任何值之前的电子邮件验证（中断流程）
- 沙箱前需要信用卡
- 多条路径“选择你自己的冒险”（决策疲劳；一条黄金路径获胜）
- API 密钥隐藏在设置中（Stripe 将它们预先填充到代码示例中）
- 无需语言切换的静态代码示例
- 将文档站点与仪表板分开（上下文切换）

## 第 2 遍：API/CLI/SDK 设计

**黄金标准：**
- **条带前缀 ID**：`ch_` 表示费用，`cus_` 表示客户。自我记录。不可能传递错误的 ID 类型。
- **Stripe 可扩展对象**：默认返回 ID 字符串。 `expand[]` 获取内联的完整对象。嵌套扩展最多 4 层。
- **条带幂等键**：在突变上传递 `Idempotency-Key` 标头。安全重试。没有“我重复收费了吗？”焦虑。
- **Stripe API 版本控制**：首先调用将帐户固定到当天的版本。通过 `Stripe-Version` 标头测试每个请求的新版本。
- **GitHub CLI**：自动检测终端与管道。在终端中可读，在管道传输时以制表符分隔。 `gh pr <tab>` 显示所有 PR 操作。
- **SwiftUI 渐进式披露**：`Button("Save") { save() }` 到完全定制，每个级别的 API 相同。
- **htmx**：HTML 属性取代 JS。总共 14KB。 __代码_0__。零构建步骤。
- **shadcn/ui**：将源代码复制到您的项目中。你拥有每一条线。没有依赖性，没有版本冲突。

**反模式：**
- Chatty API：一项用户可见的操作需要 5 次调用
- 命名不一致：`/users`（复数）与 `/user/123`（单数）与 `/create-order`（URL 中的动词）
- 隐式失败：200 OK，错误嵌套在响应正文中
- 上帝端点：47 个参数组合，每个子集具有不同的行为
- 需要文档的 API：第一次调用之前需要 3 页文档 = 太多的仪式

## 第 3 步：错误消息和调试

**三层错误质量：**

**第 1 层，Elm（会话编译器）：**
```
-- TYPE MISMATCH ---- src/Main.elm
I cannot do addition with String values like this one:
42|   "hello" + 1
     ^^^^^^^
Hint: To put strings together, use the (++) operator instead.
```
第一人称，完整的句子，准确的位置，建议的修复，进一步阅读。

**第 2 层，Rust（带注释的来源）：**
```
error[E0308]: mismatched types
 --> src/main.rs:4:20
help: consider borrowing here
  |
4 |     let name: &str = &get_name();
  |                       +
```
错误代码链接到教程。主要+次要标签。帮助部分显示精确的编辑。

**第 3 层，Stripe API（使用 doc_url 构建）：**
```json
{"error":{"type":"invalid_request_error","code":"resource_missing","message":"No such customer: 'cus_nonexistent'","param":"customer","doc_url":"https://stripe.com/docs/error-codes/resource-missing"}}
```
五个字段，零歧义。

**公式：** 发生了什么 + 为什么 + 如何修复 + 在哪里了解更多 + 导致该问题的实际值。

**反模式：** TypeScript 埋葬了“你是说吗？”在长错误链的底部。大多数可操作的信息应该首先出现。

## 第 4 关：文档和学习

**黄金标准：**
- **Stripe 文档**：三列布局（导航/内容/实时代码）。登录时注入 API 密钥。语言切换器在所有页面上持续存在。悬停以突出显示。用于浏览器内 API 调用的 Stripe Shell。构建并开源 Markdoc。在文档最终确定之前，功能不会发布。文档贡献会影响绩效评估。
- 52% 的开发人员因缺乏文档而受阻 (Postman 2023)
- 拥有世界一流文档的公司采用率增加了 2.5 倍
- “文档作为产品”：随功能一起提供或不提供功能

## 第 5 关：升级和迁移路径

**黄金标准：**
- **Next.js**：`npx @next/codemod upgrade major`。一个命令即可升级 Next.js、React、React DOM，运行所有相关的 codemod。
- **AG Grid**：v31+ 的每个版本都包含一个代码模组。
- **Stripe API 版本控制**：内部有一个代码库。每个帐户的版本固定。重大变化永远不会让您感到惊讶。
- **Martin Fowler 的管道模式**：编写小型的、可测试的转换，而不是一个单一的代码模块。
- Maven Central 中 21.9% 的重大变更未记录在案（Ochoa 等人，2021）

## 第 6 关：开发者环境和工具

**黄金标准：**

- 每 1 点 DXI 改进 = 每个开发人员每周节省 13 分钟。
- **GitHub Copilot**：任务完成速度提高 55.8%。 PR时间从9.6天缩短到2.4天。

## 克劳德·代码技能 DX 清单

在审查 Claude Code 技能、MCP 服务器或 AI 代理工具的计划时使用。

- [ ] **AskUserQuestion 设计**：每次调用一个问题。重新构建上下文（项目、分支、任务）。视觉反馈的浏览器切换。
- [ ] **状态存储**：全局 (~/.tool/) 与每个项目 ($SLUG/) 与每个会话。用于审计跟踪的仅附加 JSONL。
- [ ] **渐进式同意**：使用标记文件进行一次性提示。永远不要再问。可逆。
- [ ] **自动升级**：使用缓存+暂停退避进行版本检查。迁移脚本。内联报价。
- [ ] **技能构成**：连锁收益。检查链接。带部分跳过的内联调用。
- [ ] **错误恢复**：从失败中恢复。保留部分结果。检查站安全。
- [ ] **会话连续性**：时间线事件。压实恢复。跨会议学习。
- [ ] **有限自治**：明确的操作限制。破坏性行为强制升级。审计追踪。

# gstack-中文 目录结构说明

## 项目定位

`gstack-中文` 是一个围绕 AI 编码代理构建的技能与工具仓库。它不只是 skill 集合，还带有浏览器自动化、设计工具、评审体系、升级工具、主机适配层和大量测试。

## 顶层目录与文件

- `README.md`
  - 总入口。
- `ARCHITECTURE.md`
  - 架构说明。
- `ETHOS.md`
  - 项目方法论。
- `DESIGN.md`
  - 设计原则。
- `BROWSER.md`
  - 浏览器能力说明。
- `AGENTS.md`、`CLAUDE.md`
  - 代理协作规范。
- `CHANGELOG.md`
  - 版本记录。
- `SKILL.md`
  - 顶层 skill 入口。
- `package.json`
  - 依赖与脚本入口。
- `bin/`
  - CLI 命令入口。
- `browse/`
  - 浏览器访问与自动化核心模块。
- `design/`
  - 设计生成与对比工具。
- `extension/`
  - 浏览器扩展。
- `hosts/`
  - 不同宿主/平台适配。
- `scripts/`
  - 解析、路由、构建、分析脚本。
- `review/`
  - 代码审查模板与专家规则。
- `test/`
  - 仓库级测试。

## 目录关系

- 一级 skill 目录负责定义工作流。
- `bin/` 负责把能力暴露成命令。
- `scripts/` 负责解析、拼装、生成和校验。
- `hosts/` 负责平台适配。
- `browse/`、`design/`、`extension/` 是较重的功能模块。
- `review/` 与 `test/` 负责质量控制。
- `supabase/` 与 `gstack-upgrade/` 负责服务端支撑与升级迁移。

## 建议阅读顺序

1. `README.md`
2. `ARCHITECTURE.md`
3. `ETHOS.md`
4. `SKILL.md`
5. 目标 skill 目录下的 `SKILL.md`
6. 相关实现模块，如 `browse/`、`design/`、`scripts/`
7. `review/` 和 `test/`

## 常见工作路径

- 想改 skill：
  - 先看对应目录下的 `SKILL.md` 和 `SKILL.md.tmpl`
- 想改命令路由：
  - 看 `bin/` 与 `scripts/resolvers/`
- 想改浏览器能力：
  - 看 `browse/src/` 和 `extension/`
- 想改设计能力：
  - 看 `design/src/`
- 想改平台适配：
  - 看 `hosts/`
- 想查质量规则：
  - 看 `review/` 和 `test/`

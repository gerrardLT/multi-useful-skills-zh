# gstack-中文 目录结构说明

## 项目定位

`gstack-中文` 是一个面向 AI 编码代理的技能与工作流仓库。它把规划、设计、审查、测试、发布、浏览器自动化等能力拆成多个独立 skill，并提供配套脚本、浏览器扩展、评审模板与文档设计稿。

## 顶层目录说明

- `.github/`
  - GitHub 相关配置，例如工作流、模板或自动化配置。
- `agents/`
  - 面向代理运行环境的配置文件。
- `autoplan/`、`office-hours/`、`review/`、`ship/`、`qa/`、`cso/` 等
  - 各个核心 skill 的独立目录。
  - 一般每个目录里会有 `SKILL.md` 和可能存在的模板、脚本、辅助文件。
- `benchmark/`、`benchmark-models/`
  - 基准测试与模型对比相关 skill。
- `bin/`
  - 命令行可执行脚本与工具入口。
- `browse/`
  - 浏览器自动化与网页访问能力的核心实现。
  - 同时包含自己的 `src/`、`test/`、脚本和页面资源。
- `contrib/`
  - 贡献时可复用的补充内容，例如新增 host 的辅助模板。
- `design/`
  - 与设计生成、设计比较、设计服务相关的实现代码。
- `docs/`
  - 面向使用者和维护者的说明文档、设计稿、专题文章。
- `extension/`
  - 浏览器扩展相关代码与静态资源。
- `gstack-upgrade/`
  - 升级流程、版本迁移、升级相关 skill。
- `hosts/`
  - 各平台或宿主环境的适配层。
- `lib/`
  - 通用库代码。
- `make-pdf/`
  - PDF 生成相关能力。
- `model-overlays/`
  - 面向不同模型的补充说明或覆盖层。
- `openclaw/`
  - 与 OpenClaw 相关的接入说明和技能桥接内容。
- `plan-ceo-review/`、`plan-eng-review/`、`plan-design-review/`、`plan-devex-review/`、`plan-tune/`
  - 不同规划/审阅场景的 skill。
- `qa/`、`qa-only/`
  - 质量保障与页面测试工作流。
- `review/`
  - 代码审查、专项审查、清单模板、专家评审资料。
- `scripts/`
  - 构建、生成、同步、分析等辅助脚本。
- `supabase/`
  - Supabase 相关函数、迁移和配置。
- `test/`
  - 仓库级测试、夹具和辅助测试代码。

## 关键文件说明

- `README.md`
  - 项目总入口，适合先读。
- `AGENTS.md`
  - 面向代理协作的规则说明。
- `CLAUDE.md`
  - Claude 环境下的使用约束和行为规则。
- `ARCHITECTURE.md`
  - 项目整体架构说明。
- `BROWSER.md`
  - 浏览器能力相关说明。
- `ETHOS.md`
  - 项目理念与方法论。
- `DESIGN.md`
  - 设计与风格相关说明。
- `CHANGELOG.md`
  - 版本更新记录。
- `SKILL.md`
  - 顶层 skill 入口说明。
- `package.json`
  - Node/Bun 项目元信息与脚本入口。

## 目录关系

- `SKILL` 目录群 是项目的核心内容层。
  - 每个 skill 目录负责一个明确工作流。
- `bin/`、`scripts/`、`lib/` 是项目的工具实现层。
  - 为 skill 提供命令、脚本与共享逻辑。
- `browse/`、`design/`、`extension/` 是较重的能力模块。
  - 分别对应浏览器能力、设计能力、扩展能力。
- `docs/`、`review/`、`model-overlays/` 是说明与治理层。
  - 提供文档、评审标准、模型差异化说明。
- `test/` 与各子模块内 `test/` 共同构成验证层。

## 建议阅读顺序

1. `README.md`
2. `AGENTS.md`、`CLAUDE.md`
3. `ARCHITECTURE.md`、`ETHOS.md`
4. 目标 skill 目录下的 `SKILL.md`
5. 若涉及浏览器或设计，再看 `browse/`、`design/`、`extension/`
6. 需要深入维护时，再看 `scripts/`、`bin/`、`lib/`、`test/`

## 常见工作路径

- 想了解怎么用：
  - 先看 `README.md`，再看对应 skill 目录。
- 想改某个 skill：
  - 先看该目录的 `SKILL.md`，再看相关 `scripts/`、`bin/` 或 `lib/`。
- 想改浏览器行为：
  - 重点看 `browse/`、`extension/`、`BROWSER.md`。
- 想补文档或评审规则：
  - 重点看 `docs/`、`review/`。

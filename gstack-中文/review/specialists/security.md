# 安全专家审查清单

范围：当 SCOPE_AUTH=true OR (SCOPE_BACKEND=true AND diff > 100 行)
输出：JSON 对象，每行一个结果。架构：
{“严重性”：“严重|信息”，“信心”：N，“路径”：“文件”，“行”：N，“类别”：“安全”，“摘要”：“...”，“修复”：“...”，“指纹”：“路径：行：安全”，“专家”：“安全”}
可选：行、修复、指纹、证据、测试存根。
如果没有发现：输出 `NO FINDINGS` 而没有其他内容。

---

该清单比主要的关键阶段更深入。主代理已经检查 SQL 注入、竞争条件、LLM 信任和枚举完整性。该专家专注于 auth/authz 模式、加密滥用和攻击面扩展。

## 类别

### 信任边界的输入验证
- 未经控制器/handler 级别验证而接受用户输入
- 直接在数据库查询或文件路径中使用的查询参数
- 无需类型检查或模式验证即可接受请求正文字段
- 文件上传无需 type/size/content 验证
- 无需签名验证即可处理 Webhook 有效负载

### 身份验证和授权绕过
- 端点缺少身份验证中间件（检查路由定义）
- 授权检查默认为“允许”而不是“拒绝”
- 角色升级路径（用户可以修改自己的角色/permissions）
- 直接对象引用漏洞（用户A通过更改ID访问用户B的数据）
- 会话固定或会话劫持机会
- Token/API 密钥验证不检查过期时间

### 注入向量（SQL 之外）
- 通过使用用户控制参数的子进程调用进行命令注入
- 使用用户输入的模板注入（Jinja2、ERB、Handlebars）
- 目录查询中的 LDAP 注入
- 通过用户控制的 URL（获取、重定向、Webhook 目标）进行 SSRF
- 通过用户控制的文件路径进行路径遍历 (../../etc/passwd)
- 通过 HTTP 标头中用户控制的值进行标头注入

### 密码滥用
- 用于安全敏感操作的弱哈希算法（MD5、SHA1）
- 令牌或秘密的可预测随机性（Math.random、rand()）
- 对秘密、令牌或摘要进行非恒定时间比较 (==)
- 硬编码加密密钥或 IV
- 密码散列中缺少盐

### 秘密曝光
- 源代码中的 API 密钥、令牌或密码（甚至在注释中）
- 应用程序日志或错误消息中记录的秘密
- URL 中的凭据（URL 中的查询参数或基本身份验证）
- 返回给用户的错误响应中的敏感数据
- 当需要加密时，PII 以明文形式存储

### 通过逃生舱口的 XSS
- Rails：.html_safe，用户控制数据上的 raw()
- React：dangerouslySetInnerHTML 与用户内容
- Vue：带有用户内容的 v-html
- 姜戈：|安全，用户输入上的 mark_safe()
- 一般：带有未经净化的数据的innerHTML分配

### 反序列化
- 反序列化不受信任的数据（可执行类型的pickle、Marshal、YAML.load、JSON.parse）
- 接受来自用户输入或外部 API 的序列化对象，无需模式验证

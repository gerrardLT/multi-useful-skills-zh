# 性能专家审查清单

范围：当 SCOPE_BACKEND=true 或 SCOPE_FRONTEND=true 时
输出：JSON 对象，每行一个结果。架构：
{“严重性”：“严重|信息”，“信心”：N，“路径”：“文件”，“行”：N，“类别”：“性能”，“摘要”：“...”，“修复”：“...”，“指纹”：“路径：行：性能”，“专家”：“性能”}
可选：行、修复、指纹、证据、测试存根。
如果没有发现：输出 `NO FINDINGS` 而没有其他内容。

---

## 类别

### N+1 查询
- ActiveRecord/ORM 关联在循环中遍历，无需急切加载（.includes、joinedload、include）
- 可以批处理的迭代块（each、map、forEach）内的数据库查询
- 触发延迟加载关联的嵌套序列化器
- GraphQL 解析器查询每个字段而不是批处理（检查 DataLoader 使用情况）

### 缺少数据库索引
- 没有索引的列上的新 WHERE 子句（检查迁移文件或架构）
- 非索引列上的新 ORDER BY
- 没有复合索引的复合查询（WHERE a AND b）
- 添加没有索引的外键列

### 算法复杂度
- O(n^2) 或更糟糕的模式：集合上的嵌套循环、Array.map 内的 Array.find
- 可以使用 hash/map/set 查找的重复线性搜索
- 循环中的字符串连接（使用 join 或 StringBuilder）
- 在一次就足够的情况下多次对大型集合进行排序或过滤

### 捆绑包大小影响（前端）
- 已知的新生产依赖项（moment.js、lodash full、jquery）
- 桶导入（从“库”导入）而不是深度导入（从“库/specific”导入）
- 未经优化而提交的大型静态资产（图像、字体）
- 缺少路由级块的代码分割

### 渲染性能（前端）
- 获取瀑布：可以并行的顺序 API 调用 (Promise.all)
- 来自不稳定引用的不必要的重新渲染（渲染中的新对象/arrays）
- 在昂贵的计算中缺少 React.memo、useMemo 或 useCallback
- 在循环中读取和写入 DOM 属性导致布局抖动
- 下屏图像上缺少加载=“lazy”

### 缺少分页
- 列出返回无限结果的端点（无限制，无分页参数）
- 没有LIMIT的数据库查询随着数据量的增长而增长
- 嵌入完整嵌套对象而不是带扩展的 ID 的 API 响应

### 异步上下文中的阻塞
- 异步函数内的同步 I/O （文件读取、子进程、HTTP 请求）
- 基于事件循环的处理程序内的 time.sleep() / Thread.sleep()
- CPU 密集型计算会阻塞主线程，而无需卸载工作线程
